# 📡 Daily Pulse — 每日脉搏 / 日报目录

## 📌 目录说明

`daily-pulse/` 目录用于存放 **每日脉搏（Daily Pulse）** 报表。每日脉搏是对项目全局状态的快照采集与验收报告，涵盖 TSC（TypeScript 编译检查）、测试状态、代码审计结果以及部署健康检查。通常由自动化脚本在每日固定时间触发生成。

## 📂 已有内容索引

| 文件名 | 说明 |
|--------|------|
| `pulse-365.md` | 🦞 第 365 号验收脉冲（2026-07-12） — 包含 TSC 14/14 全缓存通过、各模块测试状态、知识库纳管结果 |

## 📋 文件命名规则

```
pulse-{序号}.md
```

- `序号` 为从 1 开始递增的脉冲编号
- 每篇脉冲文件包含：状态采集 → 详细状态 → 异常/瓶颈分析 → 认知与经验总结

## 🚀 使用指引

1. **查看最新脉搏**：打开编号最大的 `pulse-*.md` 文件即可获取最近一次系统快照
2. **回溯历史状态**：按序号浏览历史脉冲，可追踪项目健康度变化趋势
3. **自动化触发**：每日脉搏由 CI/CD 流水线或定时脚本自动生成并提交
4. **紧急问题识别**：脉冲报告中的 `⚠️` / `❌` 标记表示需关注的风险项，应及时处理

## 📝 维护说明

- 每篇脉搏应在同一自然日内生成，避免跨天混淆
- 建议保留至少 30 天的历史脉搏以备回溯
- 若自动生成失败，可手动参考 `pulse-365.md` 模板格式补录
- 脉冲标题中的 🦞 标识为 shenjiying 项目的标志性符号
