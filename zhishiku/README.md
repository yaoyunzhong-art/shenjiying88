# 📚 知识库（Zhishiku / Knowledge Base）

神机营 SaaS 领域**商业情报与行业知识**采集储存系统，涵盖亲子乐园/儿童游乐场馆的六维深度数据。

## 简介

zhishiku/ 目录（中文简称"知识库"）是神机营面向亲子游乐行业的商业情报中枢。通过自动化侦察兵（Scout Agent）按轮次（Round）执行定向采集，产出覆盖全国各省市亲子乐园场馆的多维专题数据。采集范围包括：头部连锁品牌扩店追踪、P0 六维深度补全（到店距离·停车位·月流水·经营许证·消防验收·设备故障率·团购核销率）、县级市场与价格扫描等。所有数据 **仅追加**，不删除历史记录。

## 核心概念

### 1. 侦察兵轮次（Scout Round）
每次侦察采集对应一个轮次编号（如第 158 轮、第 160 轮、第 162 轮）。轮次编号持续递增，标识数据采集的时间线和专题。

### 2. P0 六维
神机营定义的场馆核心评估体系，共六个维度（P0-1 至 P0-6），用于评估一个场馆的投资价值、运营难度和风险等级。

### 3. 仅追加无删除（Append Only）
宪法级规则——所有采集数据永久保留，不存在删除/覆盖操作。新数据只能在文件末尾追加。旧数据即使过时也不删除，仅标记状态变化。

### 4. 数据来源标注
每个数据点标注来源（如"极目新闻""企查查""抖音团购页"），确保可追溯、可验证。推测值（如意算月流水）明确标注"估算""对标""推测"等关键词，区分于已验证事实。

### 5. 采集日志
每条专题数据末尾包含标准日志格式：轮次编号、时间戳、数据来源、说明和宪法级声明。

## 使用指南

### 查阅场馆数据
```bash
cat zhishiku/national-venue-database.md
```
在文件中搜索场馆名即可定位对应六维数据。

### 搜索某个品牌
```bash
grep -A 10 "MELAND" zhishiku/national-venue-database.md
```

### 查看最新采集轮次
```bash
tail -100 zhishiku/national-venue-database.md | grep "侦察兵"
```

### 新增场馆/专题数据
1. 编辑 `national-venue-database.md`
2. 在文件末尾追加新数据
3. 标注轮次编号、时间戳和数据来源
4. 使用 `---` 分隔不同轮次区块
5. 以 `> **宪法级·仅追加无删除 ✅` 结尾

## 维护规则

- **严禁删除**：任何情况下不得删除已有数据行
- **每次追加标注轮次**：新数据必须有明确的轮次标识和时间戳
- **数据来源必填**：每个推断必须有来源标注，纯推测须注明"估算"
- **格式统一**：使用 markdown 表格对比价格和数据
- **宪法级声明**：每个轮次的数据块必须以宪法级声明结尾
- **IP归属品牌标注**：国际 IP 品牌（如小猪佩奇）标注授权方
