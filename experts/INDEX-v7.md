## 四、每位新专家细节

### E45 廖前端 — 前端架构师

**领域**: 前端架构·组件工程化·SSR/CGI/性能优化

**绑定Phase**: P-35收银前端 + P-36会员前端 + P-25~34全部前端

**关键交付**:
1. 跨7应用(admin-web/storefront-web/tob-web/app/mobile/miniapp/uis)的统一组件体系
2. 前端性能基准+监控 (LCP/FID/CLS)
3. 组件库(packages/ui)的模块化拆分/按需加载
4. 前端CI管道优化 (turbo cache + 增量构建)

**新增到专家组**: 前端工程化组

---

### E46 施运维 — SRE/可观测性专家

**领域**: SRE·可观测性(日志/指标/链路)·稳定性工程

**绑定Phase**: P-41异常检测 + P-42自愈机制 + P-53运维自动化

**@m5/api 20天hang根因诊断**:
1. NestJS Prisma 依赖链 -> 测试积压 -> 增量跳过 -> 依赖未更新
2. pnpm-lock drift -> 测试环境与生产不一致
3. Node test runner 时序竞争 -> Promise 挂起

**关键交付**:
1. 全链路可观测性: OpenTelemetry → 链路追踪/RED指标/审计日志
2. 健康检查API + 自动恢复机制
3. @m5/api 测试环境隔离(容器化, 避免Promise冲突)
4. 告警规则: 5min内自动发现+恢复

**新增到专家组**: 运维组 (与E8周运营配合)

---

### E47 谷测试 — 测试架构师

**领域**: 测试架构·自动化测试策略·质量门禁

**绑定Phase**: P-35~50全域测试

**关键交付**:
1. 测试分层策略: L1单元(100%通过→merge) / L2集成(定时跑) / L3 E2E(每2h)
2. 测试覆盖率目标: service.spec 52% → 80%, E2E 33模块全覆盖
3. 假阳性清理: role.test重建mock, 避免824 fail
4. 质量门禁: TSC绿→test绿→E2E绿→merge

**新增到专家组**: 质量管理组

---

### E48 霍数据 — 大数据/数据平台专家
