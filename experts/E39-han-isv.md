## 2. 💡 核心洞察

### 洞察 1: 开放平台 = 「给第三方开发者配钥匙」

**观点**: 开放平台的本质不是「暴露API」，而是「让第三方开发者理解你的系统、信任你的平台、愿意投入时间」。

**开放平台设计三原则**:

**原则1: 开发者体验优先**
- 开发文档: 中文+英文+代码示例（Node.js/Python/Java/Go）
- API Playground: 在线测试API（无需写代码即可体验）
- SDK包: npm/pip/maven 一键安装
- 错误信息: 中文错误码+解决方案链接

**原则2: 渐进式授权**
```
游客级:        公开API (查游戏厅位置/营业时间)
    ↓ OAuth 授权
基础开发者级:   读API (查订单/查库存/查会员)
    ↓ 应用审核
高级开发者级:   写API (创建订单/修改库存)
    ↓ 安全审计
ISV伙伴级:     管理API (创建租户/配置系统)
```

**原则3: 沙箱与生产隔离**
- 沙箱环境: 完全独立的数据库+Redis+消息队列
- 沙箱数据: 初始种子数据（测试门店/会员/商品）
- 沙箱权限: 与生产一致（API Rate Limit ×10宽松）
- 沙箱→生产: 一键部署（但需要应用审核）

### 洞察 2: OAuth 2.0 实战落地

**观点**: OAuth 2.0不是理论协议，是「授权安全工程」。

**神机营OAuth流程**:

```
┌────────────┐    ① 跳转授权页面    ┌──────────────┐
│  第三方App  │──────────────────▶│  神机营Auth  │
│  (ISV)     │                    │  Server      │
└────────────┘                    └──────────────┘
      │                                  │
      │  ② 用户确认授权                    │
      │◀──────────────────────────────────│
      │  ③ 授权码(code)                   │
      │◀──────────────────────────────────│
      │  ④ 用code换token                  │
      │──────────────────────────────────▶│
      │  ⑤ 返回access_token+refresh_token │
      │◀──────────────────────────────────│
      │  ⑥ 使用token调用API               │
      │──────────────────────────────────▶│
```

**Token生命周期**:
- `access_token`: 2小时过期（短生命周期，减少泄露风险）
- `refresh_token`: 30天过期（可续期，减少用户重新授权）
- 敏感操作: 要求re-auth（如财务API额外确认）
- Token吊销: 用户主动/管理员强制/应用下架

### 洞察 3: ISV应用商店 = 「SaaS的App Store」

**观点**: ISV应用商店不是「放几个插件的页面」，而是应用发现→试用→订购→使用的全流程。

**应用商店架构**:

| 模块 | 功能 | 责任人 |
|------|------|--------|
| 开发者中心 | 创建应用/提交审核/查看数据 | E39 韩ISV |
| 应用市场 | 浏览/搜索/对比/试用 | 租户管理员 |
| 应用管理 | 已安装/更新/卸载 | 门店管理员 |
| 计费中心 | 订购/续费/退款 | E10 郑财务 |

**审核流程**:
1. ISV 提交应用（含功能介绍+API使用声明+安全承诺）
2. 自动扫描: API权限最小化检查+代码安全扫描
3. 人工审核: 功能测试+用户体验+性能测试
4. 沙箱测试: 租户在沙箱环境体验
5. 上架: 进入应用市场（可设置免费/付费/试用）

---

### 洞察 4: SDK多语言 = 降低开发者门槛60%

**观点**: 好的SDK能消除60%的开发摩擦。劣质SDK=开发者流失。

**SDK设计规范**:
- 类型安全: TypeScript/Python type hints/Java @NotNull
- 自动重试: 指数退避（3次重试，200ms→400ms→800ms）
- 错误处理: 抛出领域异常（`StoreNotFoundError` 而非HTTP 404）
- 分页: 自动处理next_cursor（对调用方透明）
- 日志: 内置调试日志开关（生产环境默认关闭）

**语言优先级**:
1. 🥇 TypeScript (Node.js) — 主力
2. 🥈 Python — 数据分析用户
3. 🥉 Go — 高并发场景
4. 🏅 Java — 企业级客户
