# 📋 专家模板目录

> shenjiying88 体系 · M5 平台「40+4 专家团」角色模板

## 项目简介

本目录存放 **shenjiying88** 专家模块的**角色模板文件**，为 E1–E54 专家角色的创建和管理提供标准格式参考。模板定义了统一的角色描述结构，包括职责范围、领域知识、系统交互模式等字段，确保 44 个专家角色在 8 大领域及老板团队中的一致性。

## 目录结构

```
experts/templates/
├── README.md                        ← 🟢 本文件（模板目录说明）
└── expert-profile.md                ← 📄 专家角色标准模板
```

父级目录中的关键索引文件：
- `experts/INDEX.md` — 完整专家索引（E1–E54 角色总表）
- `experts/INDEX-v7.md` — 索引归档版本
- `experts/README.md` — 专家模块根目录说明

## 快速开始

1. **创建新的专家角色**：复制 `expert-profile.md` 到 `experts/` 目录，按模板填写各字段
2. **命名规范**：新文件遵循 `E<编号>-<姓氏拼音>-<领域>.md` 格式（如 E55-wang-analytics.md）
3. **同步索引**：创建新角色后，更新 `experts/INDEX.md` 添加对应条目

## 配置说明

| 文件 | 说明 |
|------|------|
| `expert-profile.md` | 专家角色标准模板，含角色名称、领域归属、职责描述、知识范围、系统交互等 |

模板字段约定：
- **角色编码**：`E<编号>` 格式，递增分配
- **领域归属**：从 8 大领域中选择（架构/安全/营销/数据/财务/体验/租户/后勤）或老板团队
- **职责范围**：清晰界定该角色的业务决策范围与知识边界
- **系统交互**：描述该角色与 MCP 平台其他模块的集成方式
- **Phase 映射**：标注该角色在 P-35/P-36/P-38 等冲刺中的参与度

## API / 模块说明

专家模板本身不提供 API，但 `expert-profile.md` 中定义的系统交互字段决定了专家角色如何通过 MCP 协议与以下模块协同：
- `docs/knowledge/` — 知识库
- `usa-street-game-mcp/` — 街机游戏配置
- `apps/` — 各应用模块

## P-38 财务冲刺

**专家模板模块**在 **P-38 财务冲刺** 中提供以下支撑：

- **财务角色标准化**：财务领域专家（E10-zheng-finance、E29-li-investor 等）均基于本目录模板创建，确保 P-38 相关的财务分析与损益模型有一致的角色定义
- **验收协作**：财务专家的模板中定义的职责范围，与 `docs/acceptance/finance-p38/acceptance-finance-p38.md` 验收标准对齐
- **模板迭代**：P-38 的财务流程变化可反馈到模板更新，形成可持续的专家角色生命周期管理
