## 专业洞察
### 关键洞察
1. **前端i18n**: 推荐react-intl + ICU消息格式。已支持16种语言包管理(提取→翻译→编译→加载)。按需加载: 用户只加载其语言。
2. **后端i18n**: 错误消息/邮件模板必须通过i18n框架，禁止硬编码中文。支持区域化: 币种(CNY/USD/HKD)、日期格式、税收政策。
3. **翻译流水线**: GitHub Actions自动提取→DeepL/Google Translate机翻→创建PR→人工校验→发布。每个语言一个JSON文件。

### 开发赋能建议
- 所有用户可见文本必须通过i18n函数
- 日期/货币/数字必须使用Intl API
- 新增语言必须通过PR审查(至少1人校验)
