# DR-019: 多模态 Embedding 分维策略

**Status**: Accepted
**Date**: Phase-23 (2026-06-26)
**Context**: Phase-23 T81 多模态 embedding 升级

## 决策

文本和图片采用**不同维度**的 embedding:
- 文本: **384 维** (MiniLM-style mock)
- 图片: **512 维** (CLIP-style mock)
- 跨模态: 切片到 min(dim) + cosine similarity

## 背景

V1 阶段只支持 256 维文本 embedding,无法处理图片等多模态输入。Phase-23 需要:
1. 文本升级到 384 维 (更精细语义)
2. 新增图片 512 维 (CLIP 风格)
3. 跨模态检索 (文搜图 / 图搜文)

## 取舍

| 方案 | 优点 | 缺点 |
|------|------|------|
| **同维度 (e.g., 384 都用)** | 简单,无需切片 | 浪费空间,语义不清 |
| **分维度 (本方案)** | 语义清晰,可独立优化 | 跨模态需投影 / 切片 |
| **统一 1024 维** | 不需跨模态处理 | 文本噪声大,图片仍不足 |

## 后果

- ✅ 文本语义清晰 (词袋 + 位置编码 + hash 噪声)
- ✅ 图片特征丰富 (尺寸 + 颜色直方图 + hash)
- ✅ 跨模态 cosine 简单可解释
- ⚠️ 跨模态检索需要 slice 对齐 (min(textDim, imageDim))
- ⚠️ mock sha256/md5 必须用正确字节数 (32/16) 避免 NaN

## 关键修复

1. **sha256 长度**: 32 字节 (不是 64),循环条件用 `i < 32`
2. **md5 长度**: 16 字节 (不是 32),循环条件用 `i < 16`
3. **零向量边界**: cosine 除零时显式返回 0,避免 NaN
