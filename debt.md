## 本轮新增债务 (Pulse-Nightly-16)

### P1-N16-001: 串行测试中的now作用域污染
- **发现**: 链25 N2/N3/B1 因 `now` 箭头函数隐式引用模块级 `now` 变量导致 ReferenceError
- **修复**: 在 `getPointBalance` 内用 `_now` 局部变量避免与外层 `const now` 冲突
- **经验**: 同一文件内函数级 `now` 引用需独立声明, 避免无意识依赖全局模块变量
- **影响等级**: 🟢 低 (已修复)

### P1-N16-002: exchangeItem 兑换计数逻辑缺陷
- **发现**: 原实现 `records.filter(r => r.totalPoints < 0).length * -1` 产生负数计数器, 导致always=success
- **修复**: 改为 `deductionRecords.length` 即可
- **经验**: 业务计数函数中的 `*-1` 反转需要谨慎; 用绝对逻辑比反转更安全
- **影响等级**: 🟢 低 (已修复)

### P1-N16-003: 月度积分上限配置与测试不一致
- **发现**: gold会员 `monthlyPointCap=5000`, 但测试硬编码1000导致断言失败
- **修复**: 改用bronze配置(实际cap=1000)重构测试场景
- **经验**: 测试中硬编码配置值必须与数据源完全一致; 建议测试引用数据源常量而非魔数
- **影响等级**: 🟢 低 (已修复)

### P1-N16-004: 链27 N3 幂等性规则条件扩展
- **发现**: 已下架商品(infra:disabled)不应再次触发下架动作, 但原始规则条件只有 `stock===0`
- **修复**: 补充 `status !== 'disabled'` AND条件
- **经验**: 规则引擎的幂等性需要用额外状态字段保护; 单条件触发可能有重复执行风险
- **影响等级**: 🟢 低 (已修复)

## 本轮新增债务 (Pulse-Nightly-15)

### P1-N15-001: 数据管道时间戳精度导致断言失效
- **发现**: 链22 P1 测试中 `syncEvent.completedAt > syncEvent.startedAt` 断言因同一毫秒时间戳而失败
- **修复**: 改为 `>=` 宽松断言
- **经验**: 同一事件循环内的操作共享相同 `Date.now()`; 跨模块 E2E 测试中时间序断言需要容忍零差
- **影响等级**: 🟢 低 (已修复)

### P1-N15-002: 订单状态机不完整导致 markDelivered 失败
- **发现**: 链23 P1 测试中 `preparing → delivered` 直接跳转未被状态机允许; 正确路径为 `preparing → shipping → delivered`
- **修复**: 补充 shipping 状态转换步骤
- **经验**: 业务状态机需要严格遵循定义; 测试必须完整走通状态转移链而非跨级跳转
- **影响等级**: 🟢 低 (已修复)

### P1-N15-003: 版本冲突场景需先播种高版本数据
- **发现**: 链22 N1 测试中传入 `sourceVersion=0` 的陈旧数据期望失败, 但因产品键 `prod-001` 尚未存在(版本优先匹配规则), 实际被当作新增成功
- **修复**: 先播种一个高版本 (v3) 记录, 再传入低版本 (v1) 触发版本落后失败
- **经验**: 版本冲突测试必须建立明确的前置状态; 空记录集下 "版本落后"无意义
- **影响等级**: 🟢 低 (已修复)

### P1-N15-004: 管理看板异常订单时间阈值需要负值跨越同一ms
- **发现**: 链23 B2 测试 `getAbnormalOrders(orders, 0)` 因所有订单在同一毫秒创建(age=0, 不满足 `> 0`) 导致0条检测
- **修复**: 用 `-1` 阈值实现同一毫秒检测
- **经验**: 毫秒级精度测试中, 严格 `>` 时间比较对同一事件循环的操作无效; 需要用 `>=` 或负阈值
- **影响等级**: 🟢 低 (已修复)

### 新增持续债务摘要 (Pulse-Nightly-15)
- **时间戳零差**: 同一事件循环内 `Date.now()` 返回相同值; 时间序断言需用 `>=` 而非 `>`
- **状态机跳转严谨**: 业务状态转换必须严格按照定义链; 跨步跳转应触发错误
- **前置状态依赖**: 负向测试(版本冲突/非法转换)需要清晰的前置条件建立

### 新增持续债务摘要 (Pulse-Nightly-16)
- **函数内now变量隔离**: 工具函数中的 `now` 引用可能无意引用外层模块变量; 需用 `let/const _now` 隔离
- **测试数据源一致**: 配置项(monthlyPointCap)应与数据源严格一致, 禁止硬编码魔数
- **规则引擎幂等性**: 条件触发动作后, 需要额外状态字段防止重复执行
- **兑换计数符号安全**: 扣减记录计数应使用 `filter.length` 而非 `filter.length * -1`
