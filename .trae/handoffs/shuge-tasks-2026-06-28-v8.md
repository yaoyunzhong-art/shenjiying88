## 📋 任务清单 (按宪法优先级 P0-P3)

### 🔴 P0 · 必做 (宪法 P0 级硬约束)

#### T-V8-P0-101 · 5 端脚手架 + 基础组件库 (Phase 52-58 阶段 1)
**宪法依据**: 第二条 5 端统一
**耗时**: 阶段 1 7 phase 持续 (主战场)
**5 端脚手架**:
1. **PC 端**: Next.js 14 (App Router) · apps/storefront-web / apps/tob-web / apps/admin-web
2. **H5 端**: Next.js 移动适配 · apps/h5-web (新建)
3. **APP 端**: React Native + Expo · apps/mobile
4. **Pad 端**: Next.js 响应式 · apps/pad-web (新建,共用 PC)
5. **小程序端**: Taro 3 · apps/miniapp (微信 + 支付宝 + 抖音 + 百度)

**M5 Design System 扩充** (阶段 1 必做):
- AI 选择题组件 (`@m5/ai-choice`)
- AI 推送组件 (`@m5/ai-push`)
- AI 知识库卡片组件 (`@m5/ai-kb-card`)
- 5 端跨端适配 (统一 token,响应式 breakpoints)
**验收**: 5 端脚手架跑通 hello-world,M5 组件库 ≥ 50 个组件

#### T-V8-P0-102 · 3 类站点前端实现 (Phase 52-65 全程)
**宪法依据**: 第二条 3 类站点
**耗时**: 14 phase 持续
**站点清单**:
1. **ToC 官网** (阶段 1 主场): C 端用户 + AI 推荐引擎前端
2. **ToB 官网** (阶段 2 主场): B 端企业 + AI 匹配引擎前端
3. **统一管理后台** (阶段 3-4 主场): 运营 + AI+知识库中枢前端
**验收**: 3 站点全部上线,接入 AI 引擎

#### T-V8-P0-103 · 4 类工作台前端实现 (Phase 66-79)
**宪法依据**: 第六条 全链路工作台
**耗时**: 阶段 3-4 14 phase 持续 (主战场)
**4 工作台清单**:
1. **W-1 ToC 前台** (阶段 4): C 端用户 + AI 推荐 + 数据看板 + 个人中心/订单/会员/AI 待办
2. **W-2 ToB 前台** (阶段 4): B 端企业 + AI 匹配 + 商机 + 合作申请/供应链/结算
3. **W-3 业务中台** (阶段 4): 运营/业务 + AI 协同 + 推送 + 跨主体协同/业务全景/异常告警
4. **W-4 管理后台** (阶段 3): 管理员 + **AI+知识库中枢** ⭐ + 内容运维/权限/租户/**AI 推送选择题**
**AI 闭环前端**:
```
AI 主动推送待办 + 选项 → 用户选择题 → 操作完成 → 日志 → 知识库沉淀
                                          ↓
                              持续优化后续 AI 引导精度
```
**验收**: 4 工作台全部上线,AI 闭环可视化

#### T-V8-P0-104 · 跨端组件 + 逻辑复用 (Phase 52-86 全程)
**宪法依据**: 第二条 跨端组件 + 逻辑复用
**耗时**: 持续 35 phase
**共享包设计**:
- `@m5/utils`: 5 端通用工具 (format/validation/i18n)
- `@m5/types`: TypeScript 类型共享
- `@m5/sdk`: API client (tRPC client)
- `@m5/domain`: 业务模型 + 状态机

- `@m5/ai-push`: AI 推送组件
**跨端逻辑**:
- tRPC (类型安全 API 调用)
- React Query (缓存)
- Zustand (轻量状态)
**验收**: 共享包覆盖率 ≥ 80%,跨端组件复用率 ≥ 70%
