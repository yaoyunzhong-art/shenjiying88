## 📐 多租户 + 数据隔离机制

### 租户级别

```
Level 1: 品牌主体 (Brand)        - 顶级租户
Level 2: 独立租户 (Tenant)      - 中级租户
Level 3: 部门 (Department)      - 内部组织
```

### 数据隔离 3 层

**L1 - 应用层 (强制)**:
- Service 层强制注入 `tenantId` (AOP 切面)
- Controller 校验 `tenantId` 与登录用户匹配
- 跨租户访问必须 Champion 审批

**L2 - 数据库层 (RLS)**:
- PostgreSQL Row-Level Security 策略
- 业务表全部加 `tenantId` 字段
- RLS policy 强制 `current_tenant_id() = tenantId`

**L3 - 资源层 (K8s)**:
- 每租户独立 namespace
- 资源配额 (CPU/内存/存储)
- 网络隔离 (NetworkPolicy)

### 跨端数据同步规则

| 场景 | 同步方式 | 延迟 | 一致性 |
|------|----------|:----:|:------:|
| 同一用户跨端 (PC↔H5) | WebSocket + Redis | < 1s | 强 |
| 跨租户协同 | 事件总线 + CDC | < 5s | 最终 |
| 跨品牌数据 | Kafka + 审批流 | < 1h | 最终 |
| 全端实时图表 | SSE + 缓存 | < 2s | 强 (读) |
