## 一、90 天冲刺原则 (大飞哥指令)

### 1.1 时间压缩可行性分析

**V9 原始计划**: 18 月 (540 天) / Phase 87-92 / 6 阶段
**V10 压缩目标**: **90 天** (压缩 6 倍)

**可行性依据**:
- 后台 + 前台 双线并行 (龙虾哥 + 树哥 各 100%)
- MVP 优先: 核心功能先上线,边缘功能后迭代

- 外部借力: 等保测评 / 渗透测试 / OpenAPI 文档站 → 第三方机构
- 减少缓冲: 每日 standup + Champion 实时评审 (0 等待)

**外部机构 (提前 30 天预约)**:
- 等保测评机构 (国家级测评中心)
- 渗透测试团队 (OWASP ZAP + 人工)
- OpenAPI 文档站托管 (SwaggerHub / 自建)

### 1.3 MVP 与增强边界

**MVP (必须 90 天上线)**:
- ✅ 大模型配置: 系统预设 + 门店自主 + 一键切换 + 历史回滚
- ✅ 付费授权: LicenseGuard + 双拦截 + 4 触发 + 自动激活

- ✅ 三级配置: W-S / W-T / W-B 3 工作台
- ✅ 权限隔离: 字段级 + 实例级 + 等保三级
- ✅ 测试验收: 180 用例 + 接口文档

**增强 (90 天后迭代)**:
- ⏳ 灰度发布 (10%→50%→100%) → 简化版 (1 次性切换 + 监控)
- ⏳ 5 语言示例 → 优先 curl + Python + JavaScript 3 语言
