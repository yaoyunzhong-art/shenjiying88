## 7. Migration Guide

无需迁移。所有 V9 既有功能保留,Sprint 1 模块以增量方式新增。

如需启用新模块,在 `apps/api/src/app.module.ts` 中导入对应 Module 即可:
```typescript

import { ReportModule } from './modules/report/report.module'
import { CanaryModule } from './modules/canary/canary.module'
```

## 8. Known Limitations

1. **Mock HTTP Server**: `e2e-flow.test.ts` 使用内存 mock,不连真实数据库。生产环境需用 `supabase start` 启动本地数据库后再跑完整 E2E。
2. **Tenant Subscription 锁定**: tenant-B listStoreConfigs('store-A1') 测试在内存模式无法直接验证,需用真实 RLS 数据库才能验证 tenant_id 隔离。
3. **OAuth rate limit 持久化**: 当前 `rateLimitByClient` 是内存 Map,生产需 Redis。
4. **License 签名**: V10 Sprint 1 License 未做 HMAC 签名,仅靠 tenant_id + license_id 索引。V10 Sprint 2 将增加 HMAC 签名防伪。
