## 7. 📌 维护规则

### 7.1 谁来维护
- 当前无 Champion,**临时由 main agent 在 Pulse 周期内维护**
- R8 通过后,由 E5 + E40 联合维护 (E5 主笔, E40 复核)
- 每个 Pulse 周期同步一次 (每日 standup 时增量更新)

### 7.2 维护内容
1. **新增 RFC**: 提交后 24h 内登记到 §2 投票历史
2. **投票更新**: 收到投票后 1h 内更新到对应 RFC 投票明细
3. **决议归档**: 投票窗口结束后 24h 内更新"决议"和"实施 Owner"
4. **任命更新**: RFC 通过后 24h 内更新到 §3/§4 任命流水
5. **评级更新**: 每月 1 日同步 §5.1 评级分布

### 7.3 归档格式
- 使用 [template.md](../../rfcs/voting/template.md) 标准格式
- 投票明细按时间倒序 (最新在上)
- 关联 RFC 文档使用相对路径

### 7.4 异常处理
- **投票未达门槛**: 标记"阻塞",启动 7 天延期
- **Champion 否决**: 立即标记"阻塞",启动 14 天申诉
- **紧急 veto**: 24h 内登记到 §4.3
