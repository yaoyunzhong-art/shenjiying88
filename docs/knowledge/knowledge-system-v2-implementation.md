## 核心概念：五环知识工厂

```
采集    →   处理   →   入库   →   赋能   →   反馈
(R1-5)     (E1-5)     (S1-6)     (A1-5)     (F1-5)
  ↓           ↓          ↓          ↓          ↓
搜索/专家   提取归类   6核心库    派单注入   使用审计
验收/竞品   权重计算   反模式库   审查阻断   退化曲线
安全扫描    赋能卡片   正向库     验收检查   ROI计算
            关联图谱   每日精华   竞品响应   老化清理
  │           │          │          │          │
  └─────── 12种科学玩法贯穿 ────────┘
```

## 各环操作细则

### R 环：知识采集 (谁·何时·产什么)

每天产生知识的 5 个源头，每个源都有明确的产出物。

```
R1 [11:00] 网络搜索 → 结构化行业洞察条目(每条约≤140字摘要+来源+标签)
R2 [08:00/14:00/20:45] 专家学习 → 专家建议卡片(含优先级/模块映射)
R3 [每脉冲] 验收反馈 → 反/正向模式条目(AM-xxx / PP-xxx)
R4 [周日04:00] 竞品数据 → 竞品动向摘要(含三战术建议)
R5 [07:30] 安全扫描 → 合规要求检查项(等保/PII)
```

### E 环：知识处理 (标准化流程)

每条采集来的原始信息必须经过以下 5 步处理才能入库：

```
E1 提取结构化: 原始文本→ {摘要(≤140字), 标签[], 来源, 时间戳, 可信度}
   → 输出: raw_knowledge_entry

E2 归类标记: raw_entry → 匹配到6大知识库之一 + 领域标签
   → 领域标签: 竞品|技术|市场|用户|合规|设备|会员|运营|选址
   → 输出: classified_entry

E3 权重计算: classified_entry → score = trust×0.30 + vetted×0.40 + fresh×0.30
   → trust: 官方=100, 行业报告=80, 自媒体=50, 用户评论=30
   → vetted: 专家检查过=100, 未检查=50
   → fresh: 今日产生=100, 每日-(100/90), 90天=0
   → 输出: scored_entry

E4 赋能卡片生成: scored_entry → "📚 [标签] 摘要 (来源, 新鲜度分)"
   → 格式: "[标签] 摘要 (来源, 评分)"
   → 例: "[竞品] 订阅制会员收入全国平均22% (ZVZO2026, 90分)"
   → 输出: empower_card (≤200字)

E5 关联图谱: scored_entry → 映射到哪些模块/Phase/PRD
   → 输出: knowledge_graph_update (写入 knowledge-graph-aware.md)
```

### S 环：知识存储 (6 个核心库 + 补充库)

```
S1 业务洞察 → competitive-intelligence.md (竞品/行业/市场/用户数据)
S2 技术决策 → adr/*.md + architecture-decisions.md (架构/模式/技术选型)
S3 合规要求 → business-rules.md + cx-constraints.md (安全/数据/法规)
S4 反模式库 → code-ringbeam-alignment.md 反模式节 (AM-xxx条目)
S5 正向模式库 → 同上正向模式节 (PP-xxx条目)
S6 每日精华 → daily-brief.md "知识KPI"栏 (当日最值得记的3-5条)
```

### A 环：知识赋能 (知识变现为行动)

**A1 派单知识注入** — 每次派单树哥时在末尾加入 `## 📚 知识赋能参考` 节

格式：
```markdown
## 📚 知识赋能参考
- [标签] 摘要 (来源, 新鲜度分)
- [标签] 摘要 (来源, 新鲜度分)
→ 引用示例/代码参考/合规检查
```

**A2 代码审查阻断** — 验收脉冲时匹配反模式库
```javascript
// 伪代码逻辑
if (newCommit.includes("as any") && antiPatternDB.has("AM-001")) {
  BLOCK("AM-001: as any 被禁用。见反模式库 AM-001")
}
```

**A3 验收知识检查** — 圈梁第三道箍的升级版
> 检查当前 commit 涉及模块是否引用了关联的最新知识？
> 如果"该模块有 3 条未读新知识" → 提示"建议阅读后提交"

**A4 能力基线** — 每周日更新"团队已知什么"
> 格式：`docs/knowledge/knowledge-baseline.md`
> 内容：当前活跃知识总数、各领域分布、最高评分TOP10、最低评分TOP10

**A5 竞品响应任务** — 竞品周更新触发
> 竞品动向→自动生成 1-3 个战术开发任务
> 例：竞品上线AI排班→生成"AI排班对标开发任务"

### F 环：知识闭环 (反馈驱动进化)

**F1 使用率审计** — 每日 22:00
统计每条知识被引用的次数、模块、时间戳。
> 存储：`docs/knowledge/usage-stats.md`
> 格式：
```markdown
## 知识使用统计 2026-07-19
| 知识卡片 | 引用次数 | 引用模块 | 最后引用 | 新鲜度 |
|:---------|:--------:|:---------|:--------:|:-----:|
| [竞品] 订阅制会员22% | 3 | P-38/dashboard | 07-19 12:00 | 90 |
| [技术] RLS 8护栏 | 1 | P-31 | 07-18 14:00 | 75 |
```

**F3 退化曲线** — 每日 22:00
```
for each 知识 in 已知知识:
  if 今天未被引用:
    知识.新鲜度 -= 10
    if 知识.新鲜度 < 50: 加入"⚠️ 知识提醒" → daily-brief
    if 知识.新鲜度 < 20: 归档到 archive/
  else:
    知识.新鲜度 = min(100, 知识.新鲜度 + 5) // 被引用时恢复
```

**F4 赋能评分** — 每日 22:00
```
每日赋能评分 = 知识引用次数 × 10 + 新增知识条数 × 5
```

**F5 知识老化清理** — 每周日 22:00
> 新鲜度 < 20 的 → 归档到 `docs/knowledge/archive/retired-knowledge-YYYY-MM-DD.md`
> 归档时保留原始内容，前面加 `[已归档 新鲜度:n]` 标记

## KPI 指标体系

| 指标 | 计算公式 | 目标 | 检查频率 |
|:-----|:---------|:----:|:--------:|
| 知识覆盖率 | 有知识的模块 / 总模块数 × 100% | > 60% | 每日 14:00 |
| 派单引用率 | 引用知识的派单 / 总派单 × 100% | > 80% | 每日 22:00 |
| 知识新鲜度 | 活跃知识平均新鲜度 | > 60 分 | 每日 22:00 |
| 知识使用率 | 当日引用的知识 / 总活跃知识 × 100% | > 30% | 每日 22:00 |
| 知识密度 | 关联知识数 / 千行代码 | > 3 条/千行 | 每周日 |
| 赋能评分 | 引用次数×10 + 新增×5 | > 50/日 | 每日 22:00 |
| 知识老化率 | 本周归档条数 / 总条数 | < 10% | 每周日 |

## 示例：一次完整的知识闭环

```
11:00 搜索到"订阅制会员收入占22%" (R1)
  → E1: 提取→ ["订阅制会员收入22%全国平均", [竞品,会员], ZVZO2026]
  → E2: 归类→ S1(业务洞察) + 标签[会员,收入模型]
  → E3: 评分 = 80×0.30 + 50×0.40 + 100×0.30 = 74分
  → E4: 卡片 → "[竞品] 订阅制会员收入全国平均22% (ZVZO2026, 74分)"
  → E5: 关联→ P-38财务模块, 会员系统模块

12:00 午间派单(P-38 dashboard增强)
  → A1: 自动检索→该卡片最匹配P-38
  → 注入派单: "## 📚 知识赋能参考\n[竞品] 订阅制会员收入全国平均22% (74分)"

14:00 树哥完成→验收时命中"A2代码阻断"否? → 无阻断
  → 日常commit通过

22:00 闭环检查:
  → F1: 该卡片今日被引用1次(12:00派单), 新增+1
  → F3: 被引用过, 新鲜度 +5 → 79分
  → F4: 赋能评分 = 1×10 + 0×5 = 10 (本卡片贡献)
```
