## 二、设计方案: 双循环自进化

```
              ┌──────────────────────┐
              │   🦞 主循环 (我执行)    │
              │  每天轮转专家组学习     │
              └──────┬───────────────┘
                     │ 触发
                     ▼
     ┌──────────────────────────────────────┐
     │  L1: 网络文献研究 → 写入专业洞察章节   │
     │  L2: 项目代码审查 → 写入学习笔记       │
     │  L3: Phase进展关联 → 更新活跃度/反馈   │
     └──────────┬───────────────────────────┘
                │ 写入
                ▼
     ┌──────────────────────────────────────┐
     │  44位专家各自的 .md 档案              │
     │  (每个档案持续增长，不可回退)           │
     │  结构: 元信息 + 专业洞察 + 学习笔记     │
     │         + 投票记录 + 反馈日志 + 进化日志│
     └──────────────────────────────────────┘
```

## 三、分时安排

每天按组轮转，确保每个专家 **每2周至少学习1次**:

| 时间 | 组 | 包含专家 | 领域 |
|:----:|:--:|:---------|:-----|
| 周一 | 架构组 | E1/E3/E39 | 基础设施 |
| 周二 | 安全组 | E2/E6/E38 | 安全合规 |
| 周三 | 业务组 | E4/E8/E11/E12/E13/E15 | 收银/会员 |
| 周四 | 数据组 | E5/E9 + E10/E29/E36 | 数据/AI/财务 |
| 周五 | 体验+租户组 | E7/E21/E40 + E26/E27/E28 | UX+多租户 |
| 周六 | 品牌组 | E30/E31/E32/E33/E34/E35 | 品牌/供应链 |
| 周日 | 后勤+管理层 | E14/E18/E20/E22/E23/E25 + Champion 5人 | 后勤/管理 |

每天处理: **5-8个专家**，每个专家产出:
1. 网络搜索当前领域最佳实践 (1-3个)
2. 项目代码审查匹配 (Phase进展→专家视角)
3. 追加到该专家的 `## 学习笔记` 章节
