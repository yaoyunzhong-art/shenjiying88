# P-31 多租户隔离 · 需求卡片
> 创建: 2026-07-14 | 截止: 7/20 (店A多租户安全前置收口)

## PRD 收口
- PRD-011: 多租户隔离

## 业务需求
P-31 不是单点接口，而是整个 SaaS 平台的隔离底座。门店、品牌、租户、市场四层上下文必须可解析、可隔离、可限额、可治理，否则后续 P-35 / P-36 / P-38 / P-49 都会建立在不稳定的租户边界上。

## 技术方案
E44 周技术 + 树哥收口口径：`apps/api/src/modules/tenant` 负责租户上下文、隔离判定、生命周期、配额与 HTTP 暴露；`apps/api/src/modules/tenant-config` 负责多级租户配置、字段脱敏、缓存与角色级访问；`docs/knowledge/p31-multi-tenant-architecture.md` 作为 RLS vs Schema 选型与运行时落点说明。

## 树哥接单清单
- `tenant.middleware + tenant.decorator + tenant.types`: 对齐 `RQ-31-01` / `AC-31-01`
- `tenant-isolation.util + tenant.service`: 对齐 `RQ-31-02` / `AC-31-01` / `AC-31-03`
- `tenant.controller + tenant-lifecycle.service + tenant-quota.service`: 对齐 `RQ-31-03`
- `tenant-multitenant.e2e + tenant-isolation.e2e + tenant.phase-p31`: 对齐 `RQ-31-04` / `AC-31-03`
- `tenant-config.controller/service/cache`: 对齐租户级配置隔离、脱敏、缓存与角色访问

## 优先顺序
1. P0: tenant header/context 注入、隔离判定、跨租户拒绝
2. P0: quota / lifecycle / CRUD HTTP 主链
3. P1: tenant-config 多级配置与敏感字段脱敏
4. P1: 迁移工具与 RLS 全表自动覆盖
