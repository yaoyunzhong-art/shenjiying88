## PRD 收口
- PRD-010: SSE后勤管理 — Logistics (P-30)

## 业务需求
P-30 的核心是“设备巡检 + 清洁排班 + 维修工单 + 物料申领”。当前仓库并非完全空白，但现有证据呈现为“两段式”：`apps/admin-web` 已有巡检页面雏形，`apps/api/src/modules/reservation` 已有可复用的时间安排/状态流转底座；两者都还没有真正长成 `PRD-010` 所要求的后勤主链。

## 树哥接单清单
- `apps/admin-web/app/stores/[id]/inspection/page.tsx`
  - 当前可对齐 `RQ-30-01 / AC-30-01 / AC-30-02` 的前端展示层
  - 已具备巡检列表、分类筛选、巡检统计、新建巡检入口、失败项“派单维修”按钮
- `apps/admin-web/app/stores/[id]/inspection/page.test.ts`
  - 当前前端静态证据（标题/筛选/统计/表格/按钮）
- `apps/api/src/modules/reservation/reservation.service.ts`
  - 当前仅可作为后勤排班/任务状态机的可复用底座
  - 已具备时间范围、状态流转、冲突检测、多租户隔离
- `apps/api/src/modules/reservation/reservation.service.test.ts`
- `apps/api/src/modules/reservation/reservation.e2e.test.ts`
  - 当前为底座级后端证据，不等于 `维修工单/清洁排班/物料申领` 已实现

## 当前缺口
- `RQ-30-01` 缺“按时提醒 / 巡检结果落库 / 已巡检状态回写”的后端链路
- `RQ-30-02` 清洁排班尚无独立模块、排班表、区域分配、考勤逻辑
- `RQ-30-03` 维修工单尚无真正的 `报修 -> 派单 -> 维修 -> 验收` 后端实现，当前只有页面按钮和通用状态机底座
- `RQ-30-04` 物料申领/审批/出库尚未看到对应模块与测试
