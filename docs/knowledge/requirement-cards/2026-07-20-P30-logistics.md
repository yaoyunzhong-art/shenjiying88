## 业务概述
P-30 后勤管理模块覆盖"设备巡检 + 清洁排班 + 维修工单 + 物料申领 + 设备维保 + 耗材采购 + 供应商管理"七大子模块，实现门店后勤全链路数字化。

## 7 个子模块功能列表

### 1️⃣ InspectionTask (设备巡检)
- **实体**: `InspectionTaskEntity` — id / equipmentId / assigneeId / scheduledAt / status(scheduled→reminded→completed) / result(status: normal|warning|fault) / inspectorId
- **Service**: createInspectionTask / sendReminder / recordInspectionResult / listInspectionTasks / getInspectionStats
- **能力**: 巡检任务创建、提醒推送、结果录入、"已巡检"状态回写、异常自动触发维修工单
- **前端**: `apps/admin-web/app/stores/[id]/inspection/page.tsx`

### 2️⃣ CleanSchedule (清洁排班)
- **实体**: `CleanScheduleEntity` — id / assigneeId / shiftName / scheduledDate / areaCode / areaName / status(scheduled→assigned→checked_in) / checkIn(cleanerId+checkedInAt)
- **Service**: createCleanSchedule / assignArea / recordCheckIn / listCleanSchedules / getCleaningStats
- **能力**: 排班创建、区域分配、考勤签到、"已到岗"状态回写

### 3️⃣ RepairOrder (维修工单)
- **实体**: `RepairOrderEntity` — id / inspectionTaskId / equipmentId / issueDescription / reporterId / assigneeId / status(open→assigned→in_progress→completed→verified) / verification(verifierId+note+verifiedAt)
- **Service**: createRepairOrder / dispatchRepair / startRepair / completeRepair / verifyRepair / listRepairOrders / getRepairStats
- **能力**: 报修→派单→维修→验收 完整四态状态链

### 4️⃣ MaterialRequest (物料申领)
- **实体**: `MaterialRequestEntity` — id / requesterId / purpose / status(pending_approval→approved→outbound) / items(MaterialRequestItem[]) / approval(approverId+note) / outbound(operatorId+warehouseCode)
- **Service**: createMaterialRequest / approve / reject / outbound / listMaterialRequests / getMaterialStats
- **能力**: 申领→审批→出库 三态流转

### 5️⃣ MaintenanceOrder (设备维保)
- **实体**: `MaintenanceOrderEntity` — id / equipmentId / issueDescription / status(pending→in_progress→pending_acceptance→completed) / acceptanceNote
- **Service**: createMaintenanceOrder / start / complete / accept / list
- **能力**: 维保工单创建到验收闭环

### 6️⃣ ProcurementRequest (耗材采购)
- **实体**: `ProcurementRequestEntity` — id / vendorName / purpose / status(draft→pending_approval→approved→rejected→ordered→received) / approval(对接P-37审批流) / orderRecord / receiveRecord
- **Service**: create / submitForApproval / approve / reject / placeOrder / receive / listProcurement
- **能力**: 采购需求→审批→下单→收货 全生命周期，对接P-37审批工单号

### 7️⃣ Supplier (供应商管理)
- **实体**: `SupplierEntity` (logistics.supplier.entity.ts) — 供应商基础信息/资质/评级
- **DTO**: `SupplierDto` (logistics.supplier.dto.ts)
- **能力**: 供应商名录管理、资质核验、合作评级
