### L1: 脉冲收尾检查点（简化版）

**不再依赖脉冲脚本自动化**，改为在 **HEARTBEAT.md** 中固化检查点：

```
🔄 L1 实时反循环（脉冲收尾部）
  - 本轮脉冲是否有新的测试fail/TSC错误？
  - 是否记录到了 snapshots/？
  - 是否有反模式新增/触发？
```

**触发**: 每次脉冲末尾龙虾哥看一眼 HEARTBEAT.md 中的L1检查点
**产出**: 有异常才写 `memory/evolution/snapshots/YYYY-MM-DD-HHMM.md`
**自动化程度**: 手动检查点（比v1方案的"脉冲代码改造"实际可行100倍）

### L2: 每轮树哥后批量回写（简化版）

**不再要求每次修复后回写**，改为每轮树哥交付后批量回写：

```
📝 L2 闭环回写（每轮树哥交付后）
  - 本轮树哥交付了什么？测试通过/TSC？
  - 有没有值得加入反/正向模式库的？
  - 追加1-2段到 evolution-log.md
```

**触发**: 每次树哥归队+龙虾哥验收时
**产出**: evolution-log.md 追加段落
**自动化程度**: 手动（但频率从"每次commit"降到"每轮树哥"）

### L3: 每日22:00自动评分（脚本化）

**终于有cron了！**

```
📊 L3 每日评分 (22:00 cron自动)
  - 由 evolution-health-check.sh 自动运行
  - 输出到 docs/knowledge/evolution/score-YYYY-MM-DD.md
  - 不依赖turbo typecheck（纯git日志检测）
```

**触发**: 每日22:00 cron `bash docs/knowledge/evolution-health-check.sh`
**产出**: score-YYYY-MM-DD.md
**自动化程度**: **100%自动化** ✅

**已创建的cron**:
```
name: v21-l3-daily-score
schedule: cron 0 22 * * * Asia/Shanghai
payload: bash docs/knowledge/evolution-health-check.sh
sessionTarget: isolated
```

### L4: 周日批处理（锁定）

```
📈 L4 周日批处理 (周日22:00)
  - 读过去7天的score文件
  - 写 evolution-weekly.md
```

**触发**: 周日22:00
**产出**: evolution-weekly.md
**自动化程度**: 半自动（读数据手动→写文件→commit）
