# 🔍 V17 每日计划审计基线

> 最后更新: 2026-07-13 11:20 · V17 自律监督体系上线

## 审计机制总览

```
🕰️ 每小时审计 (isolated, every 60min)
  ↓ 写入审计快照
  ↓ 严重偏差 → announce 告警
  ↓ 日常偏差 → 记录在案
  ↓
📊 日终审计汇总 (main systemEvent, 23:30)
  ↓ 合并全天audit
  ↓ 偏差类型统计 + 进化建议
  ↓ 写入 evolution-log + daily-brief
```

## 偏差等级

| 等级 | 颜色 | 定义 | 处理方式 |
|:----:|:----:|:----|:--------:|
| L1 | 🟢 正常 | 偏差=0 | 仅记录 |
| L2 | 🟡 警告 | 1项轻微偏差（空窗<90min/cron error 1次） | 记录+尝试自动修复 |
| L3 | 🔴 严重 | 多项偏差（空窗≥120min/cron error≥2次） | announce告警+自动修复 |
| L4 | 🔥 崩溃 | 系统级停止（空窗>4h/工作区卡死/无commit超12h） | announce告警+人工介入 |

## 审计检查项与阈值

### 1. 提交活跃度（核心指标）

| 检查项 | 阈值 | 评分 | 触发动作 |
|--------|------|:----:|----------|
| 最近提交时间 | <60min前 | 🟢 | 不处理 |
| 最近提交时间 | 60~90min前 | 🟡 | 记录+已派单则不处理 |
| 最近提交时间 | 90~120min前 | 🟡→🔴 | 触发派树哥（若未派） |
| 最近提交时间 | ≥120min前 | 🔴 | dispatch紧急任务+告警 |
| 当日总commits | <日KPI的1/4(25@h6) | 🟡 | 记录 |
| 当日总commits | <日KPI的1/6(~17@h6) | 🔴 | 告警 |

### 2. Cron 健康

| 检查项 | 阈值 | 评分 | 触发动作 |
|--------|------|:----:|----------|
| cron error数 | 0 | 🟢 | 不处理 |
| cron error数 | 1~2 | 🟡 | 记录+force re-run |
| cron error数 | ≥3 | 🔴 | 告警+逐一force run |
| 连续error同cron | ≥2次 | 🔴 | 诊断根因 |

### 3. 工作区状态

| 检查项 | 阈值 | 评分 | 触发动作 |
|--------|------|:----:|----------|
| 脏状态时长 | <30min | 🟢 | 不处理 |
| 脏状态时长 | 30~120min | 🟡 | 记录 |
| 脏状态时长 | ≥120min | 🔴 | 自动commit+告警 |

### 4. 验收脉冲

| 检查项 | 阈值 | 评分 | 触发动作 |
|--------|------|:----:|----------|
| 上次验收 | <45min前 | 🟢 | 不处理 |
| 上次验收 | 45~75min前 | 🟡 | 检查cron是否运行 |
| 上次验收 | ≥75min前 | 🔴 | force触发验收脉冲 |

### 5. 测试密度

| 检查项 | 阈值 | 评分 | 触发动作 |
|--------|------|:----:|----------|
| 当日测试总耗时 | <1h | 🟢 | 不处理 |
| 当日测试总耗时 | 1~4h | 🟡 | 记录 |
| 当日测试总耗时 | ≥4h无测试 | 🔴 | 触发段测试 |

## 审计文件路径

```
docs/knowledge/audit-trail/
├── YYYY-MM-DD.md          # 当天审计总表（合并汇总）
├── YYYY-MM-DD-evolution.md # 偏差进化记录
└── hourly/                 # 每小时的审计快照
    └── YYYY-MM-DD-hH.md
```

## 自进化循环

每次产生偏差 → 记录根因 → 调整配置 → 避免重犯

进化记录格式:
```
## 🛠️ YYYY-MM-DD HH:mm - {偏差描述}
- 严重度: L2|L3|L4
- 发现方式: 审计检查 / 人工报告
- 根因: {分析}
- 已执行修复: {操作}
- 预防建议: {建议}
- 是否已更新基线: ✅|❌
```
