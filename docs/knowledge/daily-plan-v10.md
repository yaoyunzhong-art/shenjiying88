## 零、V9的5个问题 & 不可接受的根因

> 以下问题今天每个都发生了，以后**决不允许重犯**

| # | 问题 | V9症状 | 根因 | V10根治 |

| 1 | **计划写了≠cron建了** | 专家进化cron一个没建，V9写了但`cron add`没做 | 计划文档和执行脱节 | **cron必须手动verify**，不verify视为计划不存在 |

## 五、V10如何根治每个问题（对照V9）

### 根治1：计划必须对应cron
每次建计划后执行:
```
V10的每个锚点 → 查cron list有没有
如果没有 → git禁止提交计划文档
```

### 根治2：耗时任务用isolated
- 预算>15min → isolated
- 预算≤15min → systemEvent
- 11:00日采(60min) → **isolated ✅**
- 23:00汇总(60min) → **isolated ✅**
- 其他会议(≤30min) → systemEvent

### 根治3：配置和文档必须一致
maxConcurrentRuns:
- 文档写=3 → config.get验证=3 ✅

### 根治4：定期重启
- 每12h或context>900k → gateway restart
- 每晚23:00汇总后重启 ✅

### 根治5：唯一权威计划
- V10 = 当前唯一权威
- V8/V9 = 归档（保留但不引用）
