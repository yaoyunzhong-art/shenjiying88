## 三、V10关键变更详解

### 3.1 23:00 统一日终汇总（最重要的变更）

**代替**: 之前的分散cron(09:00晨会/11:00日采/15:00午会/20:00晚会/20:45评审)
**原因**: 分散cron的systemEvent全部被主对话队列阻塞
**改为**: 23:00一个 **isolated agentTurn**，运行90min，产出所有日终文件：

```json
{
  "name": "📡 23:00 日终汇总",
  "sessionTarget": "isolated",
  "payload": {
    "kind": "agentTurn",
    "message": "【日终汇总】\n1. 全天Phase进展+commits回顾\n2. 知识库日采(6个库各追加1条)\n3. 专家洞察报告\n4. → daily-brief.md\n5. → daily-plan-vN.md\n6. git add+commit"
  }
}
```

### 3.2 预算法则硬编码进cron prompt

**每条cron的prompt格式：**

```
⏱ 预算: XXmin | 下一锚点: HH:MM | 时间槽: XXmin
✅/❌ 预算<=时间槽? → 跑/等
```

已经硬编码入所有cron，不再"期待你记得执行"。

### 3.3 资源控制

- **cron.maxConcurrentRuns = 3** (从8降下来)
- 5路树哥限3路并行
- 减少LLM堆叠爆炸

### 3.4 知识库分层(T1+T2)已上线

- 日常读: ~3.5K tokens (原来103KB)
- 归档层: 完整保留
- 11:00日采在isolated session跑，不会再阻塞

### 3.5 专家自我进化

- 不再独立cron（已删）
- 不嵌在对话里（被吞）
- **嵌入23:00日终汇总**：每日自动读取专家档+行业搜索+产出洞察报告
