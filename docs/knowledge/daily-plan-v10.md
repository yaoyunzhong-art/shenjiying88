## 零、V9的5个问题 & 不可接受的根因

> 以下问题今天每个都发生了，以后**决不允许重犯**

| # | 问题 | V9症状 | 根因 | V10根治 |
|:-:|------|--------|------|---------|
| 1 | **计划写了≠cron建了** | 专家进化cron一个没建，V9写了但`cron add`没做 | 计划文档和执行脱节 | **cron必须手动verify**，不verify视为计划不存在 |
| 2 | **systemEvent被对话吞** | 11:00日采、会议产出全部丢失 | 长对话阻塞队列 | 耗时任务(>15min)必须用**isolated agentTurn** |
| 3 | **文档和执行不一致** | 写"预算法则"但没人执行，写"maxConcurrentRuns=3"但配成8 | 改了文档没改配置 | 改文档后必须**同步修改配置和cron**，缺一不可 |
| 4 | **终端损坏后无法工作** | 12h+长会话后所有输出变图片 | 没及时重启 | **每12h或每1000 context轮转后gateway重启** |
| 5 | **多版本计划文件矛盾** | V8/V9/V10同时存在 | 没有任何一个被归档 | **V10是唯一权威，V8/V9归档** |

## 一、cron安排（V10最终版 · 和cron list一致）

### 所有会议cron：systemEvent（短任务，预算≤30min）
```
09:00 👥晨会    | 预算30min | 系统标触发时，不管对话状态
15:00 👥午会    | 预算30min | 同上，主session处理
20:00 👥晚会    | 预算30min | 同上
```

### 耗时任务cron：isolated agentTurn（独立session，不阻塞）
```
11:00 📚知识库日采 | 预算60min | ✅ isolated · 不受对话影响
23:00 📡日终汇总   | 预算60min | ✅ isolated · 统一产出daily-brief+专家洞察+V11草案
```

```

每h   🖥️资源监控   | every
每h   keepalive     | every
15min 🐜前端        | cron分散(夜间)
```

## 二、cron总额和约束

**总cron数: 24**（3会议+2isolated+5every+7夜间+7日间短锚点）

**maxConcurrentRuns = 3**（同时最多3路LLM，防资源爆炸）

**铁律**:
- 每12h自动gateway重启（防终端损坏）
- 每晚23:00汇总后关闭当前会话（防止早上堆积）

## 五、V10如何根治每个问题（对照V9）

### 根治1：计划必须对应cron
每次建计划后执行:
```
V10的每个锚点 → 查cron list有没有
如果没有 → git禁止提交计划文档
```

### 根治2：耗时任务用isolated
- 预算>15min → isolated
- 预算≤15min → systemEvent
- 11:00日采(60min) → **isolated ✅**
- 23:00汇总(60min) → **isolated ✅**
- 其他会议(≤30min) → systemEvent

### 根治3：配置和文档必须一致
maxConcurrentRuns:
- 文档写=3 → config.get验证=3 ✅

### 根治4：定期重启
- 每12h或context>900k → gateway restart
- 每晚23:00汇总后重启 ✅

### 根治5：唯一权威计划
- V10 = 当前唯一权威
- V8/V9 = 归档（保留但不引用）
