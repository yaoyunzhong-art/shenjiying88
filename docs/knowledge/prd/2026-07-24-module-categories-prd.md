# 商品分类 (Categories) — PRD 文档

## 1. 背景

### 1.1 业务背景

神机营 SaaS 零售平台中，所有商品（StockItem / POS 商品）需要统一分类体系来实现：商品归类、库存盘点分组、采购计划分类、报表按类目汇总、门店陈列规划等场景。

当前分类数据源自 POS 一线的实际商品类别，需要提供标准的 CRUD 接口供前端管理后台、运营后台多个系统调用。

### 1.2 痛点

1. 无统一分类管理模块，各子模块各自维护分类列表
2. 分类名称不标准（同一种商品在不同门店归属不同分类）
3. 分类管理纯手工，无法对接自动化盘点/采购
4. 缺乏关键词检索能力，运营人员找分类靠翻页

### 1.3 目标

- 提供标准的商品分类 CRUD API
- 支持关键词模糊搜索（名称 + 描述）
- 支持分类统计（总数、名称列表）供报表使用
- 内置种子分类数据（10 个主流分类）降低初始化成本
- 可接入商品计数，作为库存/采购/报表的基础维度

## 2. 功能需求

### 2.1 分类查询

| ID | 功能 | 描述 |
|----|------|------|
| F-01 | 全部分类列表 | `GET /categories` → 返回所有分类数组，每个含 name / description / productCount |
| F-02 | 按名称查询 | `GET /categories/:name` → 精确查找（大小写不敏感），不存在返回 404 |
| F-03 | 分类统计 | `GET /categories/stats` → 返回 `{ total, categories: string[] }` |

### 2.2 分类管理

| ID | 功能 | 描述 |
|----|------|------|
| F-04 | 创建分类 | `POST /categories` → body: `{ name, description }`，去重校验，重名返回 409 |
| F-05 | 更新分类 | `PATCH /categories/:name` → body: `{ description?, productCount? }`，部分更新 |
| F-06 | 删除分类 | `DELETE /categories/:name` → 删除指定分类，不存在返回 404 |

### 2.3 分类搜索

| ID | 功能 | 描述 |
|----|------|------|
| F-07 | 关键词搜索 | `GET /categories/search?keyword=xxx` → 搜索名称或描述包含关键词的分类（大小写不敏感） |

---

## 3. 非功能需求

| ID | 维度 | 要求 |
|----|------|------|
| NFR-01 | 响应时间 | P95 < 50ms（内存操作，无 DB） |
| NFR-02 | 多租户隔离 | TenantGuard 校验隔离 |
| NFR-03 | 输入验证 | 名称非空校验，URL 编码兼容（decodeURIComponent） |
| NFR-04 | 可恢复性 | reset() 方法可恢复到种子数据，用于测试和运维 |
| NFR-05 | 幂等性 | 相同名称的重复创建返回 409 而非静默覆盖 |

## 4. 数据模型概要

### 4.1 Category

| 字段 | 类型 | 必填 | 说明 |
|------|------|------|------|
| name | string | ✅ | 分类名称（唯一，大小写不敏感） |
| description | string | ❌ | 分类描述 |
| productCount | number | ✅ | 关联商品数量快照 |

### 4.2 种子数据 (SEED_CATEGORIES)

| name | description |
|------|-------------|
| 餐饮 | 食品、饮料、熟食及原料 |
| 服装 | 服饰、鞋帽、配饰 |
| 数码 | 电子产品、手机、电脑及配件 |
| 日用品 | 家庭清洁、个人护理、日常消耗品 |
| 娱乐 | 玩具、游戏、文体用品 |
| 饮品 | 瓶装饮料、茶饮、咖啡、酒水 |
| 零食 | 休闲零食、糖果、糕点点心 |
| 文具 | 办公用品、文具、纸张 |
| 医疗 | 药品、医疗器械、保健品 |
| 其他 | 未分类或其他商品 |

### 4.3 错误码 (CategoryErrorCode)

| 码 | HTTP 状态码 | 含义 |
|----|------------|------|
| CATEGORY_NOT_FOUND | 404 | 分类不存在 |
| CATEGORY_DUPLICATE | 409 | 分类名称重复 |
| CATEGORY_INVALID_NAME | 400 | 分类名称为空 |

---

## 5. 依赖关系

- **TenantGuard**: 多租户 RLS 守卫
- **存储层**: 当前为内存 `Map<string, Category>`，无外部数据库
- **调用方**: 前端运营后台（管理端）、数据分析报表（按分类汇总）
- **扩展性**: 后续可持久化到 PostgreSQL（继承 BaseEntity），并通过 productCount 字段与库存/商品模块关联
