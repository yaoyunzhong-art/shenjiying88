## 1. 业务背景

### 1.1 为什么要做
店A需要会员系统来维持复购率。客户到店→办会员→消费积分→下次用积分消费→粘性建立。
当前仅有基础会员数据库，前台收银无法识别/扣减积分。

### 1.2 成功标准
- 会员识别 ≤3秒（输入手机号→展示信息）
- 支付时自动应用积分/等级折扣
- 会员可在小程序查看自己的积分和等级

## 3. 需求清单

### 3.1 会员等级

| 等级 | 条件 | 权益 |
|:----|:-----|:-----|
| 🟤 普通 | 注册即享 | 积分1x |
| 🩶 银卡 | 累计消费≥500 | 积分1.2x + 95折 |
| 🟡 金卡 | 累计消费≥2000 | 积分1.5x + 9折 |
| 💎 钻石 | 累计消费≥5000 | 积分2x + 85折 + 生日礼 |

| RQ | 标题 | 优先级 | 验收标准 |
|:---|:-----|:------:|:---------|
| RQ-36-01 | 会员注册 | P0 | 输入手机号+姓名→创建会员→发送验证码 |
| RQ-36-02 | 会员查询 | P0 | 输入手机号→返回会员信息(姓名/等级/积分/余额) |
| RQ-36-03 | 等级展示 | P0 | 会员卡上显示当前等级+升级进度条 |
| RQ-36-04 | 积分累计 | P0 | 消费完成后自动增加积分(基于等级倍率) |
| RQ-36-05 | 积分扣减 | P0 | 客户选择用积分抵扣(100积分=1元) |
| RQ-36-06 | 余额充值 | P0 | 微信充值→余额增加→记录流水 |

### 3.3 非功能需求

| 需求 | 标准 |
|:-----|:-----|
| 查询响应 | <1s (Redis缓存会员信息) |
| 积分事务 | 消费+积分在同一事务 |
| 余额安全 | 支付时校验余额，防止并发透支 |

## 4. 数据模型

```typescript
interface Member {
  id: string;
  phone: string;        // 唯一
  name: string;
  level: 'regular' | 'silver' | 'gold' | 'diamond';
  points: number;       // 当前积分
  balance: number;      // 预付余额(分)
  totalSpent: number;   // 累计消费
  createdAt: Date;
  expiredAt?: Date;     // 会员有效期
}

interface PointsTransaction {
  id: string;
  memberId: string;
  type: 'earn' | 'redeem' | 'expire' | 'admin';
  amount: number;
  orderId?: string;
  remark: string;
  createdAt: Date;
}

interface BalanceTransaction {
  id: string;
  memberId: string;
  type: 'recharge' | 'payment' | 'refund' | 'admin';
  amount: number;
  orderId?: string;
  paymentMethod?: string;
  createdAt: Date;
}
```

## 5. 验收卡

| 卡ID | 场景 | 前置 | 预期 |
|:----|:-----|:-----|:-----|
| AC-36-01 | 注册: 输入新手机号→注册成功 | 手机号未注册 | 返回member对象+等级=普通 |
| AC-36-02 | 查会员: 输入已注册手机号 | 会员存在 | 返回姓名/等级/积分/余额 |
| AC-36-03 | 查会员: 输入未注册手机号 | 无此会员 | 提示"该手机号未注册" |
| AC-36-04 | 积分累计: 消费100元→普通会员得100积分 | 会员等级=普通 | 积分+100 |
| AC-36-05 | 积分累计: 消费100元→金卡得150积分 | 会员等级=金卡 | 积分+150 |
| AC-36-06 | 积分扣减: 客户选择用500积分→抵扣5元 | 积分≥500 | 积分-500, 应付-5 |
| AC-36-07 | 积分不足: 积分<100无法抵扣 | 积分<100 | 提示"积分不足" |
| AC-36-08 | 充值100元→余额+100 | 微信支付成功 | 余额增加, 流水记录 |
| AC-36-09 | 余额支付: 余额≥应付→支付成功 | 余额足够 | 余额扣减, 订单完成 |
| AC-36-10 | 余额不足: 余额<应付→提示"余额不足" | 余额不够 | 提示其他支付方式 |
