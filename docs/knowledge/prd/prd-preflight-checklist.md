# PRD 预备清单（Gate 0）

> 用途: 在龙虾哥正式派单前，把“树哥马上能开工”的 PRD 必需信息一次补齐。
> 适用范围: 新功能、跨端联动、性能优化、缓存改造、I18n 接入、官网/营销类页面。

## 1. 立项最小集

| 项 | 必填内容 | 说明 |
|:---|:---------|:-----|
| 目标 | 为什么做 | 一句话说清业务目标和收益 |
| 用户 | 谁在用 | 角色、场景、触发动作 |
| 范围 | 做到哪 | 本次包含什么、不包含什么 |
| 影响面 | 改哪些端 | API / admin-web / storefront / miniapp / docs |
| 风险 | 最怕什么 | 性能回退、租户串数据、SEO受损、文案歧义等 |

## 2. 派单前必须齐的结构

1. 标题、状态、发布日期、关联 Phase
2. 业务背景或问题定义
3. `RQ-` 需求卡，至少 3 条
4. `AC-` 验收卡，至少 3 条
5. 数据模型或接口草图
6. 非目标范围说明

## 3. 树哥开工所需补充

| 项 | 示例 | 目的 |
|:---|:-----|:-----|
| 文件落点 | `apps/api/src/modules/tenant-config/*` | 避免树哥先花时间找目录 |
| 约束 | P99、租户隔离、I18n 策略 | 防止实现跑偏 |
| 兼容性 | 是否影响现有页面/接口 | 便于补测试 |
| 验证方式 | 单测、页面测试、脚本校验 | 保证能闭环 |
| 文案归因 | 页面标题、按钮、提示词 | 减少返工 |

## 4. V17 项目专项检查

### 性能 / 缓存类

- 是否声明 P99、CPU、缓存命中率目标
- 是否声明 tenant 级缓存隔离
- 是否声明失效策略和回退策略

### I18n 类

- 是否声明默认语言与支持语言来源
- 是否说明 fallback 顺序
- 是否覆盖 portal / storefront / miniapp 展示语义

### 官网 / SEO / GEO 类

- 是否声明目标转化动作
- 是否声明页面级 meta / schema / sitemap 要求
- 是否声明多终端可访问与首屏性能要求

- `scripts/prd-validate.sh` 能过
