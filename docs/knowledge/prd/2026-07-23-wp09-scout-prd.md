## 1. 背景与目标

根据《规划6-8》第11章，同城 AI 侦察兵是五大赛道中 "知己知彼" 的核心板块。

### 核心价值
- 门店经营不再是盲打，通过竞品数据驱动战术决策
- 让价格策略从"凭经验"升级为"看数据摆事实"
- 让店长通过「商战决策书」每周知道该打谁、怎么打、多少钱

### 六大 BS 覆盖策略

| BS | 名称 | 核心内容 | 模块归属 |
|:--:|:-----|---------|:--------:|
| BS-0130 | 同城侦察兵 | 3/5km 双模式自动采集、频控、多模态 | scout/ |
| BS-0131 | 三种竞品战术 | 防守/对等/进攻 + 店长否决 | scout/ (新建) |
| BS-0132 | AI 动态定价 | 竞品价格驱动定价建议 | price-monitor/ |
| BS-0133 | 竞品 9 维数据库 | 多维度竞品信息结构化存储 | competitor-track/ |
| BS-0134 | AI 侦察周报 | 自动生成 + 推送 | scout/ (新建) |
| BS-0135 | 侦察兵知识库 | 自动更新 → 知识库 RAG | scout/ + WP-06 |

## 3. 差距分析 (BS by BS)

### BS-0130 · 同城侦察兵 (3/5km 双模式)

**要求**: 3/5km 范围可切换, 竞品公开数据自动采集, 多模态视觉, 频控≤3次/天/竞品

**已实现**: ❌
- scout/ 服务端代码是 raw SQL 查询壳子, 无采集逻辑
- 无地理位置范围计算 (GeoJSON / PostGIS)
- 无频控计数器
- 无多模态能力
- 数据库表未在 Prisma schema 中定义

**需要**: scout-city 管理, venue 表单, 采集日志, 范围参数, 频控拦截, 合规脱敏

### BS-0131 · 三种竞品战术

**要求**: 防守/对等/进攻 + 每日《商战决策书》+ 店长一键否决

**已实现**: ❌
- 完全不存在战术模型、决策书生成、否决流程
- competitor-track 只有 CRUD + 静态对比

**需要**: 全新 `scout-tactic/` 模块或扩展 scout/ → tactic 实体、决策书模板、否决路由

### BS-0132 · AI 动态定价

**要求**: 竞品数据驱动, AI 生成价格建议

**已实现**: 🟡 局部
- price-monitor 有市场均价对比 + 异常检测
- ❌ 无 AI 驱动
- ❌ 无与 competitor-track 联动
- ❌ 无定价建议/优化/模拟

**需要**: 扩展 price-monitor 服务: AI 定价引擎 (接入 WP-03A)、价格区间建议、竞争对等策略

### BS-0133 · 竞品 9 维数据库

**要求**: 9 个维度的竞品数据

**已实现**: 🟡 局部 (r18 标记 "已基本完成" 但实际不完整)
- competitor-track 有 4 维 (分类/价格水平/评分/客流)
- 缺少 5 维: 人均消费、设备明细、热⻔项目、装修年份、营业面积、营业时间、主推活动、员工数、线上评分

**需要**: 扩展 entity 到 9 维, 数据持久化到 Prisma, 数据录入/采集接口

### BS-0134 · AI 侦察周报

**要求**: 自动生成 + 推送

**已实现**: ❌

**需要**: 周报生成引擎、模板基于战术/定价建议、推送机制 (WP-13A 联动)、按门店/品牌差异化

### BS-0135 · 侦察兵知识库

**要求**: 自动更新至知识库

**已实现**: ❌
- scout 与 WP-06 知识库无连接
- 无自动更新事件/调度

**需要**: 侦察结果事件 (采集完成 / 战术生成 / 周报产出) → RAG 知识库写入

### 4.1 新增/扩展 Prisma 模型

```prisma
// scout_cities — 侦察城市配置
model ScoutCity {
  id              String    @id @default(cuid())
  name            String
  tier            String?   // 一线/二线/三线
  region          String?   // 华东/华北/华南/...
  priority        Int       @default(0)
  status          String    @default("active")  // active/inactive/error
  lastCollected   DateTime?
  nextCollect     DateTime? // 下次采集时间
  collectionLogs  ScoutCollectionLog[]
  venues          Venue[]
  createdAt       DateTime  @default(now())
  updatedAt       DateTime  @updatedAt
}

// venues — 竞品场馆 (9 维)
model Venue {
  id              Int         @id @default(autoincrement())
  name            String
  city            String
  address         String?
  lat             Float?      // 纬度
  lng             Float?      // 经度
  category        String      // arcade/game/sports/entertainment
  // === 9 维 ===
  priceLevel      Int?        // 价格水平 1-5
  avgSpend        Float?      // 人均消费
  rating          Float?      // 线上评分 1-5
  visitorCount    Int?        // 月均客流
  deviceCount     Int?        // 设备数量
  hotItems        String[]    // 热门项目
  yearBuilt       Int?        // 装修/开业年份
  area            Float?      // 营业面积 (m²)
  businessHours   String?     // 营业时间 "09:00-23:00"
  mainPromotion   String?     // 主推活动
  employeeCount   Int?        // 员工人数
  // ===
  advantage       String?     // 优势
  weakness        String?     // 劣势
  cityRelation    ScoutCity   @relation(fields: [cityId], references: [id])
  cityId          String
  prices          CompetitorPrice[]
  devices         CompetitorDevice[]
  memberships     CompetitorMembership[]
  reviews         CompetitorReview[]
  activities      CompetitorActivity[]
  createdAt       DateTime    @default(now())
  updatedAt       DateTime    @updatedAt
}

// competitor_prices — 竞品价格采集
model CompetitorPrice {
  id          Int       @id @default(autoincrement())
  venueId     Int
  venue       Venue     @relation(fields: [venueId], references: [id])
  itemName    String
  price       Float
  capturedAt  DateTime  @default(now())
}

// competitor_devices — 竞品设备信息
model CompetitorDevice {
  id          Int       @id @default(autoincrement())
  venueId     Int
  venue       Venue     @relation(fields: [venueId], references: [id])
  deviceName  String
  quantity    Int
  status      String?   // 良好/维修/停用
}

// competitor_membership — 竞品会员体系
model CompetitorMembership {
  id          Int       @id @default(autoincrement())
  venueId     Int
  venue       Venue     @relation(fields: [venueId], references: [id])
  tierName    String
  price       Float
  benefits    String?
}

// competitor_reviews — 竞品用户评价
model CompetitorReview {
  id          Int       @id @default(autoincrement())
  venueId     Int
  venue       Venue     @relation(fields: [venueId], references: [id])
  content     String?
  sentiment   String?   // positive/negative/neutral
  postedAt    DateTime
}

// competitor_activities — 竞品活动
model CompetitorActivity {
  id          Int       @id @default(autoincrement())
  venueId     Int
  venue       Venue     @relation(fields: [venueId], references: [id])
  title       String
  description String?
  startDate   DateTime
  endDate     DateTime?
}

// scout_collection_logs — 采集日志
model ScoutCollectionLog {
  id          Int       @id @default(autoincrement())
  cityId      String?
  city        ScoutCity? @relation(fields: [cityId], references: [id])
  source      String    // api/manual/vision
  status      String    // success/error
  itemsCount  Int?
  errorMsg    String?
  createdAt   DateTime  @default(now())
}

// scout_tactics — 战术决策
model ScoutTactic {
  id            String   @id @default(cuid())
  city          String
  venueId       Int?
  opponentId    Int?     // 目标竞品
  tacticType    String   // defense/parity/attack
  reason        String?  // AI 生成原因
  content       String?  // 决策书内容 Markdown
  storeVeto     Boolean  @default(false)  // 店长否决
  vetoReason    String?  // 否决原因
  status        String   @default("draft") // draft/confirmed/executed/archived
  generatedAt   DateTime @default(now())
  confirmedAt   DateTime?
  executedAt    DateTime?
  weekLabel     String?  // "2026-W30"
}

// scout_reports — 周报
model ScoutReport {
  id            String   @id @default(cuid())
  city          String
  weekLabel     String   // "2026-W30"
  reportType    String   // weekly/monthly
  summary       String?  // Markdown 摘要
  aiContent     String?  // AI 生成正文
  pdfUrl        String?
  status        String   @default("draft") // draft/published
  generatedAt   DateTime @default(now())
  publishedAt   DateTime?
}
```

## 5. 关键风险

| 风险 | 等级 | 缓解 |
|------|:----:|------|
| Prisma 表不存在, scout/ 全量 raw SQL 会失败 | 🔴 | 先建迁移 → 再改服务从 queryRaw 切到 Prisma client |
| competitor-track / price-monitor 全 In-memory | 🔴 | 全部迁移到 Prisma, 用 migration 落地 |
| AI 定价需要 WP-03A AI 中台就绪 | 🟡 | 先用规则引擎兜底 (竞品均价 ±20%), AI 接入留接口 |
| 每日采集 ≤3 次频控需要 Redis 计数 | 🟡 | Redis 现有, 复用频控模块 |
| 知识库更新依赖 WP-06 接口定义 | 🟡 | 先写适配器壳, 事件桥接 |
| 店长否决需要前端/通知链路 | 🟡 | 先实现后端否决 API, 前端后续接入 |
