---

## 1. 业务背景

### 1.1 为什么要做
店A前台收银是客户进店第一触点。收银慢=客户排队=差评=流失。
当前测试版收银台只支持"手动输入金额"，不支持会员/套餐/优惠券/支付的完整闭环。

### 1.2 成功标准
- 收银员从"客户到台"到"支付完成" ≤30秒
- 支持3种支付方式: 微信扫码/会员余额/现金
- 错误率 <1%

## 3. 需求清单

### 3.1 核心流程

```
客户到台
  ↓
收银员: 选择商品/套餐
  ↓
收银员: 确认金额
  ↓
收银员: 询问会员 → 输入手机号查会员
  ↓
系统: 自动应用会员折扣
  ↓
收银员: 选择支付方式 → 客户付款
  ↓
系统: 支付成功 → 打印小票 → 增加积分
```

### 3.2 功能需求

| RQ | 标题 | 优先级 | 验收标准 |
|:---|:-----|:------:|:---------|
| RQ-35-01 | 商品选择 | P0 | 搜索/扫码选择商品，支持多件 |
| RQ-35-02 | 套餐选择 | P0 | 展示可用套餐，选择后自动展开子项 |
| RQ-35-03 | 金额计算 | P0 | 商品金额+数量=小计，自动合计 |
| RQ-35-04 | 会员识别 | P0 | 输入手机号→查会员→显示等级+积分 |
| RQ-35-05 | 会员折扣 | P0 | 根据会员等级自动打折（金卡9折/银卡95折/普卡无） |
| RQ-35-06 | 优惠券 | P1 | 输入优惠码/选择可用优惠券，自动抵扣 |
| RQ-35-07 | 支付选择 | P0 | 微信扫码/会员余额/现金 三种选项 |
| RQ-35-08 | 支付处理 | P0 | 调用支付API，成功后更新订单状态 |
| RQ-35-09 | 小票打印 | P1 | 支付成功后自动打印（或显示打印按钮） |
| RQ-35-10 | 积分累计 | P1 | 支付成功后自动增加会员积分 |

### 3.3 非功能需求

| 需求 | 标准 |
|:-----|:-----|
| 响应时间 | 点击→响应 <500ms |
| 支付超时 | 扫码支付等待 <120s → 自动取消 |
| 可用性 | 99.5% (每月掉线 <3.6h) |
| 数据一致性 | 支付成功必须写订单+更新库存 在同一事务 |

## 5. 验收卡（自动生成·验证脚本使用）

每条验收卡对应一个自动化测试：

| 卡ID | 场景 | 前置 | 预期 |
|:----|:-----|:-----|:-----|
| AC-35-01 | 商品搜索: 输入"射击" → 展示"射击体验" | 收银台打开 | 展示≥1条商品 |
| AC-35-02 | 商品添加: 点击商品→添加到已选清单 | 商品列表展示 | 已选清单更新 |
| AC-35-03 | 多件商品: 选择射击x2 → 小计=60¥ | 添加2件射击(单价30) | 小计显示¥60 |
| AC-35-04 | 会员识别: 输入已注册手机号→展示会员信息 | 会员数据库有该手机号 | 展示姓名/等级/积分 |
| AC-35-05 | 会员折扣: 金卡9折 → 100¥折后90¥ | 会员识别完成 | 应付=¥90 |
| AC-35-06 | 优惠券输入: 输入合法优惠码→抵扣 | 会员识别完成 | 优惠券字段更新 |
| AC-35-07 | 微信扫码支付: 选择→生成支付二维码 | 确认应付金额 | 二维码生成 |
| AC-35-08 | 支付成功: 收到支付回调→显示成功页 | 支付API返回success | 订单生成+积分增加 |
| AC-35-09 | 取消订单: 倒计时结束→自动取消 | 扫码支付120s未完成 | 订单状态=cancelled |
| AC-35-10 | 空结算: 未选商品点击结账→提示"请添加商品" | 收银台打开·无商品 | toast提示 |

## 6. 数据模型

```typescript
// 收银订单
interface CashierOrder {
  id: string;           // ORD-yyyyMMdd-xxxx
  items: OrderItem[];   // 商品明细
  memberId?: string;    // 会员ID
  subtotal: number;     // 原价小计
  discount: number;     // 折扣金额
  couponId?: string;    // 优惠券ID
  total: number;        // 应付金额
  payment: PaymentType; // wechat | balance | cash
  status: OrderStatus;  // pending | paid | cancelled
  createdAt: Date;
}

interface OrderItem {
  productId: string;
  name: string;
  unitPrice: number;
  quantity: number;
}
```
