### 圈梁执行细则（V8-V22教训硬化 + 54专家审计硬化）

1. **无跳过原则**（来源V18）：
   - 任何测试不得使用 `describe.skip` `it.only` `as any`
   - 无法通过的测试修复或删除，不做"基准跳过"

2. **24h硬线原则**（来源V20）：
   - Phase截止 = TSC + 测试 + E2E 三件套通过
   - 仅"代码写完"不关Phase

3. **真实数据原则**（来源V18+V22+G2审计）：
   - G2安全审查要看**实战扫描结果**，不是签署率
   - Security baseline **真有8/8才算8/8**（G2审计发现实际仅3/8）
   - `AuthGuard @UseGuards`覆盖率需实时扫描（Gate6审计发现182/189缺显式注解）

4. **5分钟审查原则**（来源V16）：
   - Gate签署最多5分钟看摘要卡
   - 超时不签自动上报大飞哥

5. **机制配套原则**（来源V21）：
   - 新制度必须有脚本+cron+检查点
   - Gate6条件：远程推送禁令必须有**自动化检测脚本+CI门禁**

6. **E2E链分级原则**（来源G3审计）：
   - L0: 5条生命线smoke test（收银/支付/退款/对账/租户登录）
   - L1: 核心业务链（20条，每日必绿）
   - L2: 全量链（50条，每周全绿）
   - 分级上报——L0 fail → 立即阻断

7. **性能门禁原则**（来源V18+G4审计）：
   - admin-web新页面上线前LCP必须 < 2s
   - CDN/nginx缓存策略必须有配置文档
