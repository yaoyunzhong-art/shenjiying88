### 圈梁执行细则（V8-V22教训硬化）

1. **无跳过原则**（来源V18：662个fail被"跳过"）：
   - 任何测试不得使用 `describe.skip` `it.only` `as any`
   - 无法通过的测试应当修复或删除，不做"基准跳过"

2. **24h硬线原则**（来源V20：P-38最后一公里拖延）：
   - Phase截止 = TSC + 测试 + E2E 三件套通过
   - 仅"代码写完"不关Phase

3. **真实数据原则**（来源V18：签署率≠安全）：
   - G2安全审查不仅要签署，还要看实战扫描结果
   - Security baseline 8/8通过加扫描日志

4. **5分钟审查原则**（来源V16：54专家不参与）：
   - Gate签署最多5分钟看摘要卡
   - 超时不签自动上报大飞哥

5. **机制配套原则**（来源V21：写文档不建脚本=0%）：
   - 任何新制度必须配套脚本+cron+检查点
   - 不单独写"应该做xxx"——写"xxx脚本/cron已部署"
