## 2. 当前前提

- `LYT 电玩城` 与 `数字运动潮玩馆管理系统` 的真实接口文件暂未到位。
- 因此所有依赖真实 `LYT` 协议的任务, 当前必须区分:
  - `可先做`: 适配器边界、配置模型、容错降级、Mock/Real 插槽、签名接口壳。
  - `被阻塞`: 真实字段映射、签名算法、错误码协议、Webhook 语义、轮询/回调细节。
- 所有这类任务统一标记: `blocked by missing LYT api spec`。

## 3. 优先级规则

- `P0`: 不完成就不能宣称符合《规划6-8》最小可运行标准。
- `P1`: 核心业务与中台能力, 必须在主线可运行后尽快补齐。
- `P2`: 产品化、生态化、体验增强。
- `P3`: 便捷化改造、长期优化、标准化沉淀。
