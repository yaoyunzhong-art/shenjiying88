## 2️⃣ 核心问题解决方案

### 问题1: 科学开发流程没有
**方案**: 5 Phase 全流程
Phase 0 需求 → 需求卡(E3签合理+E1签可行) → PRD
Phase 1 设计 → 接口设计+测试方案(E5数据+E7体验)
Phase 2 开发 → 树哥写代码(AI预审+小粒度commit)
Phase 3 测试 → L1单元+L2角色+E2E+专家手测
Phase 4 验收 → 6道Gate手动签(5min超时自动过)

### 问题2: 专家不碰代码
**方案**: 
- 代码抽查: E1每周2次随机抽3个TS文件打分A/B/C/D, C=退修 D=重写
- Gate手动签: 6门改为5分钟手动确认(超时自动过,不阻塞)
- 角色手测: 每个Phase至少1位对口专家手动跑1个场景

### 问题3: 报告没人看
**方案**:
- 专家cron已经改为"决策卡片"模式(非散文报告)
- 每卡三要素: 一句话结论 + 发现的问题(1-3个) + 行动卡片
- 08:00晨学 / 14:00午学 已生效, 明早第一份卡片产出

### 问题4: 测试全是AI写的
**方案**:
- 测试专家矩阵: 8角色×54专家映射
- 每个模块至少由1位对口专家手动跑1个核心场景
- P-35→E13李收银, P-36→E40杨客户, 以此类推

### 问题5: 缺乏自进化
**方案**:
- 问题溯源闭环: 发现→分类→根因→修复→反模式记录→模式库→下次预防
- 每日23:00自动检查: 反模式自查+cron健康+计划偏差→V(N+1)改进
- 6项检查自动扫描: TSC/require/XSS/测试覆盖/代码抽查/Gate

### 问题6: A/B选项(15P0还是4核心)
**方案**: 选B
- 前3天先修"人流程"(代码抽查/Gate手测/需求卡/角色手测)
- 再修4个代码P0(A1持久化/B1回滚/C2所有权/E1前端集成)
- 其他11项排期到店A开业后
