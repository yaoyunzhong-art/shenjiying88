# 薪资核算流程

> 分类: HR/财务 | 标签: 薪资核算,工资发放,薪酬管理 | 适用: HR,财务,门店经理

## 概述

电玩城的薪资核算因其复杂的用工模式（正式员工+兼职/小时工+节假日加班+绩效奖金+提成）而具挑战性。一个标准门店通常涉及3-4种不同的薪酬结构：固定岗位（店长/值班经理）采用"底薪+绩效奖金"模式、一线岗位（导玩员/收银员）采用"底薪+提成+全勤奖"模式、技术岗位采用"底薪+技能津贴"模式、兼职/小时工采用"按小时计薪"模式。薪资核算的准确性不仅关系到员工满意度（薪资错误是投诉Top 3原因之一），也直接影响企业合规风险和税务成本。

一套清晰的薪资核算流程应包含五大环节：考勤确认→绩效数据采集→薪资计算→审批→发放，每个环节有明确的时间节点、责任人和交叉验证机制。建议使用专业薪资系统（如用友薪福社/薪人薪事/飞书People）替代传统Excel核算，减少人为错误。

## 核心内容

- **考勤数据确认（每月1-2日）**: 核对各岗位月考勤汇总表（排班表 vs 实际打卡记录），处理加班/调休/请假/旷工等异常考勤。正式员工采用标准工时制或综合计算工时制，兼职按实际小时核算（需确认是否超过法定月工时上限）。考勤差异须在2日前处理完毕，由门店店长和员工双方确认签字。

- **绩效数据采集（每月2-3日）**: 收集当月各岗位绩效数据——营收数据（POS系统导出个人业绩/团队业绩）、服务质量评分（SQS得分/顾客评价数据）、提成数据（会员开卡数/彩票兑换额/充值促销达成率）、其他扣减项（迟到/早退/服务违规/设备人为损坏）。所有绩效数据需有系统记录或书面依据作为支撑。

- **薪资计算（每月3-5日）**: 按岗位薪资结构分别计算：固定部分=底薪+岗位津贴+工龄工资；可变部分=绩效奖金×绩效系数+提成收入；补贴=餐补+交通补+住房补（如有）+夜班补贴（如有）；扣减项=社保公积金个人部分+个税+考勤扣款+其他扣款。使用薪资系统可自动关联考勤、绩效、社保等数据，一键生成《薪资核算明细表》。

- **审批流程（每月6-7日）**: 一级审批（店长确认）——确认薪资数据和绩效数据准确无误；二级审批（HR复核）——核算正确性、社保公积金基数、个税计算、合规检查；三级审批（财务审核/区域经理批准）——预算内审核、银行代发文件准备。如使用飞书/钉钉审批流，各级审批人可在手机端完成签字审批。

- **薪资发放与记录（每月8-10日）**: 通过银行代发系统发放，生成《银行代发明细表》并与银行确认到账。发放后48小时内开放薪资条查询（电子薪资条/HR系统/飞书小程序），员工可查看明细。每月薪资发放后需归档：薪资表、审批记录、银行回单完整保存，纸质版保存≥3年，电子版保存≥10年。

## 实际案例（基于电玩城场景）

福州某电玩城曾因使用Excel手工核算薪资，在计算一名兼职学生的工时时将"工作6天合计37小时"误算为"37天"，导致当月薪资虚高逾万元。虽然次月发现并追回，但严重影响了财务信誉和员工信任。门店随后导入薪人薪事系统，自动关联打卡记录、工时计算和薪资规则，核算时间从原来的3天缩短至4小时，错误率降为零。

## 可落地清单

- [ ] 建立薪资核算月度时间表（1日考勤→3日绩效→5日计算→7日审批→10日发放）

- [ ] 每季度进行一次薪资核算内部审计，抽查3个月薪资数据的准确性
