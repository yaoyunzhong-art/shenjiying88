## 👥 40人专家团参与机制

> 每天3轮专家评审（09晨会/15午会/20晚会），每模块3位专家（1技术+1业务+1用户/监督）

### 专家参与方式

#### 事前（开发前）— 09晨会技术层评审
- 技术层10人评审今日测试计划
- Gate1检查：架构合规、安全基线、测试策略
- 每日输出 → `docs/knowledge/expert-team/morning-review-YYYY-MM-DD.md`

#### 事中（开发中）— 所有树哥产出必经2位专家审查
- 每个 service.spec 文件产生后：对应模块的 **技术专家** 审查代码质量
- 每个 controller.spec / page.test 产生后：**业务专家** 审查业务完整性
- 树哥 commit 消息需标注「已过专家审查」（或由龙虾哥验证后标注）

#### 事后（开发后）— 15午会+20晚会评审签署
- 15:00 业务层14人+用户层10人 → Gate2+Gate4检查
  - 输出 → `docs/knowledge/expert-team/afternoon-review-YYYY-MM-DD.md`
- 20:00 全体40人+监督层6人 → 6道门汇总签署
  - 输出 → `docs/knowledge/expert-team/evening-signoff-YYYY-MM-DD.md`
  - S-01至S-06 每位专家签字
  - 结论：全部签署 / 有条件签署 / 退回X个模块
