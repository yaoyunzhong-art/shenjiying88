## 一、Module 分层设计

每个API模块遵循标准的四层架构：

```
Module (模块注册)
  │
  ├── Entity (数据模型 + Prisma schema)
  ├── DTO (数据传输对象 + 校验)
  │
  ├── Service (业务逻辑层)
  │     ├── 核心业务方法
  │     ├── 数据存取 (Prisma Client)
  │     └── 外部服务调用
  │
  ├── Controller (API 路由层)
  │     ├── RESTful 端点
  │     ├── 参数校验
  │     └── 响应格式统一
  │
  └── Module (NestJS @Module 注册)
```

## 二、圈梁对齐机制

```
第一道箍：PRD 定义         what
第二道箍：代码实现         how
第三道箍：测试覆盖         prove
第四道箍：审计检查         audit
```

**对齐状态标记**:
- 🟢 完整: 四道箍齐全
- 🟡 中等: 缺一道 (通常缺审计)
- 🔴 断裂: 缺两道以上 (无PRD+无测试)

## 三、验收脉冲流水线

```
30min周期:
  git pull → TSC类型检查 → 全量测试 → 缓存状态标注 → 报告输出
  ├── TSC零错误 = Base ✅
  ├── Service测试全绿
  ├── Controller测试全绿
  └── 角色测试全绿
```

**P0-FIRE机制**: fail→派树哥→下轮验收→闭环/重派/升级
