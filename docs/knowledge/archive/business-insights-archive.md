## 1. 门店SaaS多租户架构最佳实践

### 1.1 架构选型对比

| 模式 | 隔离粒度 | 适用场景 | shenjiying88 选择 |
|------|---------|---------|:----------------:|
| 数据库独立 (Database-per-Tenant) | ⭐⭐⭐⭐⭐ | 大型客户/合规敏感 | ❌ 成本高 |
| Schema独立 (Schema-per-Tenant) | ⭐⭐⭐⭐ | 中型客户 | ❌ 迁移复杂 |
| 逻辑隔离 (Shared Table + tenant_id) | ⭐⭐⭐ | 海量小微客户 | ✅ 当前选择 |
| 混合模式 (Hybrid) | 灵活 | 大小客户混合 | ✅ P0演进方向 |

### 1.2 shenjiying88 当前架构（已验证）

**核心模型**: Shared Table + tenant_id 策略 (shenjiying88 已验证可行)

```
┌─────────────┐     ┌──────────────┐     ┌──────────────┐
│ API Gateway  │────→│ 多租户中间件   │────→│   共享数据库    │
│ (JWT解析)    │     │ (tenant_id    │     │ (tenant_id     │
│  tenant_id   │     │  自动注入)    │     │  索引覆盖)     │
└─────────────┘     └──────────────┘     └──────────────┘
```

**关键实现**:
- **租户上下文**: 所有请求经网关解析 JWT → 提取 tenant_id → 注入请求上下文
- **数据隔离**: 核心表以 tenant_id 为第一分区键,所有查询强制带 tenant_id 过滤
- **索引策略**: (tenant_id, created_at) 复合索引覆盖绝大多数查询

| 趋势 | 说明 | 优先级 |
|------|------|:------:|
| **分库分表+租户路由** | 当单库超500租户或单表超2亿行,按租户id范围/哈希分片 | ⭐⭐⭐⭐⭐ |
| **租户级连接池** | 每个租户独立连接池上限,防止"烂租户"拖垮全站 | ⭐⭐⭐⭐ |

| **智能降级** | 非核心租户自动降级到只读缓存,保障核心租户 | ⭐⭐⭐ |
| **AI辅助迁移** | 按租户活跃度/数据量 AI 自动建议隔离升级 | ⭐⭐⭐ |

### 1.4 关键约束

```

  1. 所有 API 请求自动注入(不可靠客户端传入)
```

### 3.2 盲盒合规设计Checklist

```
✅ 概率公开：每个SKU的获得概率明确展示(精确到小数点后2位)

✅ 公示位置：购买页主流程可见,不得藏于二级页面
✅ 未成年人保护：单日消费限额(建议¥200上限)·家长监控
✅ 防沉迷提示：弹窗提醒·连续抽取间隔提示
✅ 退款机制：未拆封盲盒支持7天无理由退货(仅限线上购买)
✅ 记录留存：购买记录+概率抽取记录保留≥3年(合规审计)
✅ 概率变更通知：调整概率需提前30天公示
```

### 4.2 核心设计模式

| 模式 | 说明 | shenjiying88 场景 |
|------|------|:----------------:|
| **CQRS** | 读写分离,查询走物化视图 | 📊 报表/库存看板 |

## 1️⃣ 门店SaaS多租户架构最佳实践

### 三大主流隔离方案

| 隔离级别 | 实现方式 | 优点 | 缺点 | 适用场景 |
|---------|---------|------|------|---------|
| **独立数据库** | 每租户独立DB实例 | 隔离最强, 安全高 | 成本高, 维护复杂 | 金融/医疗等行业 |
| **共享DB+独立Schema** | 共享DB但独立Schema/DB | 隔离较好, 成本适中 | 非所有DB良好支持 | B2B中等租户量级 |
| **共享DB+共享Schema** | 同表通过 tenant_id 区分 | 成本低, 易扩展 | 隔离弱, 代码侵入高 | 2C/小型团队工具 |

### 2026实战关键洞察

- **混合隔离架构**：对大客户用独立DB，中等租户用独立Schema，小租户用行级隔离 — 平衡成本与安全
- **租户分层策略**：按"高频低数据量""低频高数据量""高频高数据量"三类标签化分类，差异化资源配置
- **弹性资源调度**：基于租户画像的"预测式调度+实时调整"，核心租户资源利用率从30%→65%
- **插件化架构**：核心功能标准化 + 定制需求插件化，定制响应从2周缩短至3天
- **分层灰度发布**：测试租户→10%普通租户→50%→核心大客户，一键回滚+租户隔离

### Go实现关键点

- 行级隔离 + GORM Scopes 自动注入 tenant_id 条件查询
- HTTP Middleware 实现租户认证(JWT)和配额控制
- Redis原子操作(Lua脚本)实现分布式配额管理

## 3️⃣ 盲盒营销与监管合规

### 法规核心要点（市场监管总局《盲盒经营行为规范指引（试行）》2023）

1. **概率公示强制**：必须公示商品种类、样式、抽取规则、商品分布、限量投放数量、抽取概率

3. **保底机制鼓励**：鼓励设定抽取时间/金额上限/次数上限，引导理性消费

5. **记录留存**：抽取概率设定、结果抽取完整记录留存不少于3年
6. **禁止品类**：药品、医疗器械、活体动物、有毒有害物品不得盲盒销售

- 抖音、拼多多等平台风控阈值调高
- 概率公示不得"模糊文学"，隐藏款概率必须明确标识
