## 一、当前自进化机制的6个根本缺陷

### 缺陷1: 自进化是"流程"而非"系统"
现状: 每天23:00做一个checklist（计划偏差→cron健康→反模式自查→产出V(N+1)）
问题: checklist 是人工条目，不是数据驱动。坏处: 不同天的检查粒度不同，容易遗漏。

### 缺陷2: 没有量化指标
现状: 全靠感觉/记忆判断"今天够不够好"
问题: 代码量、测试故障率、树哥违规率、模块覆盖率都是定性描述，不是KPI趋势

### 缺陷3: 自进化频率太低
现状: 每天1次(23:00)
问题: 一天的错误要等到23:00才分析，错过了24h全自动的关键修复窗口

### 缺陷4: 自进化与执行分离
现状: 23:00由龙虾哥手动做进化分析
问题: 进化是一个连续过程，不是晚上的一个独立任务。应该在每次异常发生时触发迷你进化

### 缺陷5: 没有闭环度量
现状: "今天做了进化"就算完成
问题: 无法衡量"进化之后比进化之前好了多少"。没有命中率/改善幅度指标

### 缺陷6: 没有自愈率追踪
现状: 记录错误但不追踪修复率
问题: 无法区分"学会了"和"只是记下来了"

## 二、V21 自进化科学化方案

### 核心变革: 从"23:00的一次checklist"到"四层持续进化引擎"

```
┌─────────────────────────────────────────────────────────┐
│               🔬 四层持续进化引擎                          │
├─────────────────────────────────────────────────────────┤
│  L1 实时反循环 (每次10min脉冲触发)                          │
│  L2 每次修复后闭环 (树哥+龙虾哥回写)                        │
│  L3 每日23:00趋势总结 (替代旧checklist)                     │
│  L4 每周日进化批处理 (全量数据跑分+策略调优)                  │
└─────────────────────────────────────────────────────────┘
```

### L1: 实时反循环（每次10min验收脉冲附带）

**触发**: 每次验收脉冲末尾自动执行
**输入**: 本次脉冲发现的所有异常（测试fail、TSC错误、树哥违规）
**处理**:
1. 异常分类（类型+模块+根因标签）
2. 查反模式库是否有匹配（已有的直接引用，新增的临时记录）
3. 如果同一模块连续3次脉冲同类型fail → 自动升级到L3
**产出**: `memory/evolution/snapshot-YYYY-MM-DD-HHMM.md`

### L2: 每次修复后闭环回写

**触发**: 树哥或龙虾哥修复一个bug/违规后
**处理**:
1. 修复的bug标题和内容
2. 根因分析(1-2句)
3. 可重复执行性评分(1-5)
4. 如果要记入反模式/正向模式库，标注"+AM"或"+PP"
**产出**: 只更新 evolution-log.md，不额外创建文件

### L3: 每日23:00趋势总结（替代旧checklist）

**输入**: L1产生的所有snapshot + L2修复记录
**产出**:
1. **当日健康评分** (0-100):
   - 测试通过率 = 当日测试pass / (pass+fail) × 40分
   - 树哥违规率 = (1 - 违规模块/总交付模块) × 20分
   - 代码质量 = (1 - TSCfail/总文件) × 20分
   - 闭环率 = 当日修复完成/当日发现异常 × 20分
2. **趋势变化**: 与昨日相比三项核心指标↑↓
3. **反模式动效**: 哪些反模式今天出现/消失/重复
4. **V(N+1)计划改进**: 基于数据的建议
5. **自动产出**: `docs/knowledge/daily-brief.md` 追加进化摘要

### L4: 每周日进化批处理

**触发**: 每周日 23:00自动
**输入**: 过去7天的L3产出
**处理**:
1. 全量数据跑分（7天平均值、峰值、谷值）
2. 反模式频率统计（哪些反模式出现最多？）
3. 树哥各agent的个体评分（合规率/交付质量）
4. 策略调优建议：
   - 哪些反模式需要新增圈梁箍
   - 哪些正向模式可以固化到MEMORY.md
   - 派单模板是否需要改进
**产出**: `docs/knowledge/evolution-weekly.md`

## 三、V21 健康评分计算公式

### 日评分 (0-100)

```
日评分 = 
  测试通过分(40) + 
  树哥合规分(20) + 
  TSC稳定分(20) + 
  闭环率分(20)

测试通过分 = min(40, (当日通过测试数 / 最大预期每日测试数 × 40))
树哥合规分 = 20 × (1 - 违规次数 / max(1, 交付模块数))
TSC稳定分 = 20 × (1 - TSC错误文件数 / 总文件数)
闭环率分 = 20 × (修复模块数 / max(1, 发现异常的模块数))
```

### 周评分 (0-100)

```
周评分 = 
  日评分均值(60) + 
  趋势奖励/惩罚(20) + 
  新模式贡献(20)

趋势奖励 = 如果周中健康评分呈上升趋势 → +10～20
趋势惩罚 = 如果周中健康评分呈下降趋势 → -10～-20
新模式贡献 = 新增正向模式数 × 5 - 新增反模式数 × 5
```

### 评分阈值
| 评级 | 范围 | 含义 | 行动 |
|:----:|:----:|:-----|:-----|
| 🟢 S | 95-100 | 优秀 | 维持 |
| 🟢 A | 80-94 | 良好 | 微调 |
| 🟡 B | 60-79 | 一般 | 需改进 |
| 🟠 C | 40-59 | 差 | 干预 |
| 🔴 D | 0-39 | 临界 | 立即检查 |

## 四、触发的自动升级路径

```
[异常发生]
    │
    ▼
┌──────────────────┐   L1 实时反循环
│ 分类异常          │   记录到 evolution/snapshot
│ 查反模式库        │
│ 连续3次同类fail？  │
└──────┬───────────┘
       │
       是否连续3次？
       ├── 否 ──→ 仅记录snapshot
       │
       ▼ 是
┌──────────────────┐   L3 升级
│ 自动创建改善cron   │    设定"修正此类"为次日晨会主题
│ 通知龙虾哥        │
└──────────────────┘
```
