## 三、龙虾哥 & 树哥 标准化协作流程

### 3.1 角色定义（参考MEMORY.md三级体系）

| 角色 | 职责 | 禁止 |
|:-----|:-----|:-----|
| 👑 大飞哥 | 提需求·围观·决策 | 不动手 |
| 🦞 龙虾哥 | 规划·PRD·评审·验收·调度·审计·提交 | 不写业务代码 |
| 🐜 树哥 | 写代码·跑测试·修bug | 不规划·不决策·不跨模块 |

### 3.2 标准化开发流水线

每个功能从需求到上线经历 **7个Gate**：

```
Gate 0: 需求确认
  └ 输入: 大飞哥/专家提出的需求
  └ 产出: PRD文档 (RQ需求卡 + AC验收卡)
  └ 负责人: 🦞 龙虾哥
  └ 耗时: 5~15min
  └ 标准: PRD模板 → 必含RQ+AC+数据模型
  
Gate 1: 方案设计
  └ 输入: PRD文档
  └ 产出: API设计/数据模型/组件树
  └ 负责人: 🦞 龙虾哥
  └ 耗时: 5min
  └ 标准: 同Phase已有设计模式

Gate 2: 任务派发
  └ 输入: PRD + 设计方案
  └ 产出: 树哥任务指令
  └ 负责人: 🦞 龙虾哥
  └ 耗时: 2min
  └ 标准: 任务指令包含PRD链接 + 验收卡

Gate 3: 代码实现
  └ 输入: 任务指令 + PRD
  └ 产出: 代码 commit (entity→service→controller)
  └ 负责人: 🐜 树哥
  └ 耗时: 5~10min/模块
  └ 标准: TSC 0 fail + 测试通过

Gate 4: PRD验收
  └ 输入: PRD + 树哥产出
  └ 产出: 验收报告 (AC通过/不通过)
  └ 负责人: 🦞 龙虾哥
  └ 耗时: 2min
  └ 标准: AC-01~AC-10 全部通过

Gate 5: 脉冲验收
  └ 输入: 树哥产出
  └ 产出: TSC 0 + 测试全绿 + PRD验证通过
  └ 负责人: 🦞 龙虾哥 (验收脉冲cron)
  └ 耗时: auto (30min)
  └ 标准: 验收脉冲所有检查项绿色

Gate 6: 日终签署
  └ 输入: 全天所有Gate记录
  └ 产出: 签署报告 + 日终审计
  └ 负责人: 🦞 龙虾哥
  └ 耗时: auto (23:00~23:30)
  └ 标准: 所有Gate闭环 → 写入evolution-log
```

### 3.3 输入输出速查表

| 流程节点 | 🦞 输入 | 🦞 产出 | 🐜 输入 | 🐜 产出 |
|:---------|:--------|:--------|:--------|:--------|
| Gate 0 需求 | 大飞哥一句话 | PRD.md | — | — |
| Gate 1 设计 | PRD.md | 技术方案 | — | — |
| Gate 2 派发 | 设计方案 | 树哥指令 | — | — |
| Gate 3 编码 | — | 验收 | 树哥指令+PRD | 代码commit |
| Gate 4 验收 | PRD+代码 | AC报告 | — | — |
| Gate 5 脉冲 | 系统自动 | 全绿报告 | — | — |
| Gate 6 签署 | 全天记录 | evolution-log | — | — |

### 3.4 协作铁律

```
🦞 龙虾哥:
  ✅ 派单前必须写PRD（gate0）
  ✅ 验收时对照PRD的AC卡
  ✅ 日终检查所有Gate是否闭环
  ❌ 不写业务代码
  ❌ 不直接修bug（那是树哥的事）
  ❌ 不跳过gate0直接派单

🐜 树哥:
  ✅ 开发前先读PRD
  ✅ 按PRD的AC卡实现功能
  ✅ 5~10min以内完成模块
  ❌ 不跳过需求直接写代码
  ❌ 不跨模块（单模块铁律）
  ❌ 不规划决策
```
