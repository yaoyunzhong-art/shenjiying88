## 六、全自动保障机制

### 熔断升级 (54人专家评估)

```bash
# 同源fail处理
×1 → 正常派树哥修复
×2 → P1(标记审查，中断非阻塞)
×3 → P0(熔断! 暂停流水线+通知+先修再继续)

# 不同源fail
连续 → 仅标记，不升级P0

# 安全扫描失败
任意一次SAST/secret/依赖扫描fail → ⛔ 硬阻断部署
```

### 质量门禁 (新增)

```yaml
quality-gates:
  - id: QG-001
    name: "覆盖率不降级"
    action: "break build"
  - id: QG-002
    name: "零新假阳"
    action: "自动派单修复"
  - id: QG-003
    name: "测试深度合规"
    action: "打回补充"
```

### 失败重试

```bash
验收fail → 分析根因 → 派树哥修复
下轮验收 → 闭环/重派/升级
同源fail ×3 → P0停线
```
