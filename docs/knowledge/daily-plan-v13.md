## 二、🔥 V13关键改进（基于07-11实证数据）

### 2.1 TSC质量门禁

**V12问题**: 122个TSC error在批量大页面后爆发，靠`@ts-nocheck`掩盖

**V13解法**:
```
每次commit前: 
  自动执行: pnpm -F @m5/types typecheck (如type-only变更)
  可选执行: pnpm -F @m5/sdk typecheck (如sdk变更)
  30min验收前: 跑全库TSC
  验收fail标记: 不提交+分析原因修复
```

### 2.2 龙虾哥专属时段（admin-web）

**V12问题**: 树哥admin-web 100%丢失率（所有树哥写的admin-web页面最终由龙虾哥重写）

**V13固定分配**:
```
龙虾哥独占 admin-web (21:00-22:00 高强度产出):
  · 平均产出: 15页/小时, 35KB/小时
  · 目标: 每天5-10个新页面
  · 复用模板: 类型+常量→Mock→列定义→详情→统计→主页面

树哥产出去向:
  · apps/api/ 模块后端 (entity+service+controller+test)
  · apps/storefront-web/ 前端 (可产出, 139页面累计)
  · scripts/ 工具脚本
  · infra/ 基础设施
```

### 2.3 验收脉冲修复

**V12问题**: 30min验收`lastRunStatus=error`（疑似超时/资源竞争）

**V13修复**:
```
· 验收统一用小模型(deepseek-chat)而非大模型
· 超时从180s降到120s
· 验收前先跑`git status`确认工作区干净
· 验收中发现fail→分析根因→派树哥→下轮验收
```

### 2.4 Git Push阻塞

**V12问题**: token属于`alex19710324`，repo在`yaoyunzhong-art`，push被403拒绝

**V13解法**:
```
· 等待大飞哥更新.git/config中的token
· 或切换到SSH key认证
· 不影响本地产出，影响远程备份
· push失败时本地commit积累，每天至少尝试一次
```

### 2.5 从测试驱动转向功能驱动

**V12测试/功能比**:
```
树哥产出: 6,157行测试 + 6,000行后端业务 + 12,000行AI模块
龙虾哥: 35,000行admin-web页面
总计: ~59K 功能代码 vs ~6K 测试代码 = 90%功能

向前进:
· 店A 19天后上线，功能欠的很多
· 测试覆盖率: ~17K pass/0 fail已经足够
· 下一个开发重点: P-31多租户 + P-35/P-36前端
```
