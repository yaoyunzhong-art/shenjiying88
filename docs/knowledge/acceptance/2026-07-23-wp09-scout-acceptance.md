## 2. 逐条验收标准

### AC-BS-0130 · 同城侦察兵

#### AC-0130-01 · 3km/5km 双模式范围切换
- [ ] `scout/venues` 端点支持 `range` 参数: `?range=3` 或 `?range=5`
- [ ] 范围以门店所在位置为中心 (lat/lng)
- [ ] 返回值中标注距离

#### AC-0130-02 · 侦察城市配置管理
- [ ] `scout/cities` CRUD 可用
- [ ] 每个城市可配置 tier / region / priority / status
- [ ] 城市状态包含: active / inactive / error
- [ ] 每次采集后更新 `last_collected` 和 `next_collect`

#### AC-0130-03 · 竞品数据采集
- [ ] `scout/collection-logs` 记录每次采集结果
- [ ] 采集日志记录: 来源 (api/manual/vision)、成功/失败、数量

#### AC-0130-04 · 采集频率控制
- [ ] 同一竞品每 24 小时内最多采集 3 次
- [ ] 频控用 Redis 原子计数实现
- [ ] 超限时返回 429 状态码 + `Retry-After` 头部

#### AC-0130-05 · 合规限制
- [ ] 仅采集公开信息
- [ ] 采集源标注清晰 (来源 URL / 渠道)
- [ ] 数据脱敏: 不存储个人身份信息

### AC-BS-0131 · 三种竞品战术

#### AC-0131-01 · 战术类型定义
- [ ] 三种战术枚举:
  - `defense`: 防守 → 低于竞品 5-10% 定价
  - `parity`: 对等 → 匹配竞品定价强调服务差异
  - `attack`: 进攻 → 大幅优惠 + 活动引流抢占份额
- [ ] 战术类型与竞品数据关联分析

#### AC-0131-02 · 战术生成
- [ ] `POST /scout/tactics/generate` 根据竞品数据生成战术建议
- [ ] 返回战术类型 + AI 生成原因 + 建议执行策略

#### AC-0131-03 · 店长否决
- [ ] `PATCH /scout/tactics/:id/veto` 店长一键否决
- [ ] 否决需记录原因
- [ ] 否决后状态变为 `vetoed`, AI 重新生成

#### AC-0131-04 · 商战决策书
- [ ] `GET /scout/tactics/:id/decision-book` 返回 Markdown 格式决策书
- [ ] 决策书包含: 竞品概况 → 战术类型 → 建议价格 → 活动建议 → 预期效果
