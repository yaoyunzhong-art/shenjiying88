# 商品分类 (Categories) — 验收文档

## 1. 验收范围

本验收涵盖商品分类模块 (`apps/api/src/modules/categories/`) 的全部功能入口，包括：

- **分类查询**：全部分类列表 (`GET /`)、按名称查询 (`GET /:name`)
- **分类统计**：分类总数与名称列表 (`GET /stats`)
- **分类创建**：创建新分类 (`POST /`)
- **分类更新**：部分更新分类描述/商品计数 (`PATCH /:name`)
- **分类删除**：删除分类 (`DELETE /:name`)
- **关键词搜索**：按名称 or 描述模糊搜索 (`GET /search?keyword=`)
- **错误处理**：重复创建、空名称、不存在删除

验收环境：staging · 默认加载 10 个种子分类

## 4. 边界场景

### 4.1 分类创建边界

| 场景 | 预期行为 |
|------|---------|
| 名称含前后空格 | 自动 trim，存储去空格后的名称 |
| 名称大小写不同 "DIANQI" vs "电器" | 视为不同分类，除非 equalsIgnoreCase 匹配 |
| description 为空字符串 | 存储为空字符串，不报错 |
| 连续创建同名分类（删除后再创） | 允许，因为删除后原条目已移除 |

### 4.2 分类删除边界

| 场景 | 预期行为 |
|------|---------|
| 删除后立即查询 | 返回 404 |
| 删除后重新创建同名分类 | 成功创建（类似 "恢复"） |
| 删除种子分类 "餐饮" | 后续列表不包含，不影响其他分类 |

### 4.3 关键词搜索边界

| 场景 | 预期行为 |
|------|---------|
| 关键词匹配名称和描述 | 所有匹配项均返回 |
| 关键词为空字符串 | 返回空数组 `[]` |
| 关键词为纯空格 | 返回空数组 `[]` |
| 关键词部分匹配 "食" | 返回 "餐饮"、"零食" 等 |

### 4.4 URL 编码边界

| 场景 | 预期行为 |
|------|---------|
| `:name` 直接传中文 "餐饮" | decodeURIComponent 后正常匹配 |
| `:name` 传 encodeURI 后的 "%E9%A4%90%E9%A5%AE" | decode 后正常匹配 |
| `:name` 传特殊字符 "%" | decode 异常导致 400 或 404 |
