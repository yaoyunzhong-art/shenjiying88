## 一、模块概述

工作流引擎模块提供审批工作流与自动化流程配置能力，支撑采购审批、报销审批、合同审批等企业流程场景。

### 1.1 核心能力

| 能力 | 说明 | 优先级 |
|:----|:-----|:------:|
| 流程拓扑展示 | 可视化呈现 `开始 → 审批节点 → 条件判断 → 执行 → 结束` 流程链路 | P0 |
| 节点类型定义 | 支持6种节点: 开始/结束/审批/条件/动作/等待 | P0 |
| 审批策略配置 | 任意一人通过 (any) / 全部通过 (all) / 金额分支路由 | P0 |
| 条件分支路由 | 金额 > ¥10,000 自动跳高级审批 | P0 |
| 驳回处理策略 | 驳回即终止流程 | P1 |
| 流程状态管理 | 当前版本、发布状态概览 | P1 |
| 三态UI | loading / empty / error 状态覆盖 | P0 |

### 1.2 模块文件清单

| 文件 | 类型 | 说明 |
|:-----|:-----|:------|
| page.tsx | Page Component | 工作流配置页面 (Next.js Client Component) |
| page.test.tsx | Test | 页面组件测试 |
| loading.tsx | Loading UI | 加载态 (Next.js App Router 约定) |

### 1.3 节点类型

| 节点 | 图标 | 说明 | 数量限制 |
|:----|:----|:-----|:--------|
| 开始 | 🟢 | 流程起点, 每个工作流有且仅有一个 | **1** |
| 结束 | 🔴 | 流程终点, 可能包含驳回结束 | ≥1 |
| 审批 | 👤 | 指定审批人, 支持 any/all 策略 | ≥0 |
| 条件 | 🔀 | 条件判断分支, 根据表达式路由 | ≥0 |
| 动作 | ⚡ | 自动化执行节点, 触发系统操作 | ≥0 |
| 等待 | ⏳ | 定时等待节点, 超时自动推进 | ≥0 |

## 二、核心实体

### 2.1 流程配置

| 字段 | 值 | 说明 |
|:-----|:---|:------|
| 当前版本 | v1 | 版本管理 |
| 流程状态 | 已发布 | 发布生命周期 |
| 审批策略 | 任意一人通过 (any) | 审批模式 |
| 条件分支 | 金额 > ¥10,000 → 高级审批 | 条件路由规则 |
| 驳回处理 | 驳回即终止流程 | 驳回策略 |

### 2.2 流程拓扑 (示例: 采购审批)

```
🟢 开始 → 经理审批 → 金额判断 → 执行处理 → 🔴 结束
                              ↘ 驳回 → 🔴 结束
```

- 金额 ≥ ¥10,000 进入高级审批
- 任意审批人驳回 → 流程立即终止

## 五、风险点

| 风险 | 影响 | 严重度 | 缓解措施 |
|:-----|:-----|:------:|:---------|
| 纯前端展示无后端 | 流程配置无法持久化、无法动态创建/编辑流程 | **高** | Phase-17+ 需配套后端 workflow engine (API → DB 持久化) |
| 无动态流程引擎 | 当前仅为示例拓扑, 未实现前端节点拖拽编排 | **高** | 需引入 react-flow / xyflow 等流程画布库 |
| 无审批人实际对接 | 审批节点示意但无实际审批人选择/通知 | **中** | 需对接 member/notification 模块 |
| 条件分支仅写死 | 金额 > ¥10,000 为示例值, 不可配置 | **中** | 需设计可配置条件规则引擎 |
| 无历史版本管理 | 流程当前版本 v1 不可回滚/对比 | **低** | 后续补充 versioning 能力 |
| 缺少 E2E 测试 | page.test.tsx 存在但未验证完整流程渲染 | **低** | 补充 Playwright 快照测试 |
