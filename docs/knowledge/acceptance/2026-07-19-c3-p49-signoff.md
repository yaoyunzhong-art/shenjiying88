## 当前指标面

| 指标维度 | 当前口径 | 验收标准 |
|----------|----------|----------|
| 租户隔离 | 所有密钥、配置、调用结果必须按 `tenantId` 隔离 | 任一测试不得出现跨租户读写成功 |
| 鉴权链 | OAuth2、API Key、HMAC 签名三链并存 | 合法请求成功，非法请求返回拒绝态 |
| 调用质量 | 成功率、失败率、时延、token 用量、费用估算 | `tenant-llm` 统计结构完整，字段非空 |
| 文档输出 | 对外 API 文档与合同必须可导出 | `openapi/open-api` 文档导出链有测试覆盖 |
| Webhook 可靠性 | 重试、失败、死信链可验证 | 重试与失败分支必须可被断言 |
| Sandbox / Usage | 沙箱调用与额度计量可回放 | sandbox/usage 核心路径可执行 |

---

## 调用质量签收标准

1. `OAuth2 client_credentials` 成功和失败路径都可验证。
2. `HMAC` 签名必须校验合法、过期、篡改三类路径。
3. `API Key` 必须覆盖创建、列表、撤销、校验。
4. `Webhook` 必须覆盖发送成功、失败、重试或降级口径。
5. `tenant-llm` 必须覆盖多租户隔离、状态流转、配额剩余、统计计算。
6. 核心模块的圈梁测试需在本地可重复执行，不依赖人工点击。
