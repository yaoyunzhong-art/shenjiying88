## 1. 目标

### 1.1 业务目标
建立店A完整会员体系，覆盖从注册→消费→积分→等级→权益的闭环，提升客户复购率与粘性。

### 1.2 产品目标
- 前台收银可通过手机号3秒内识别会员
- 支付时自动应用折扣与积分
- 会员可在小程序端自助查看积分、等级、余额、消费记录
- 余额充值/支付全流程线上化

### 1.3 成功指标
| 指标 | 目标值 | 测量方式 |
|:----|:------|:---------|
| 会员识别时间 | ≤3秒(含Redis缓存) | 接口P99 |
| 会员注册转化率 | ≥进店客户60% | 注册数÷进店客户数 |
| 余额使用率 | ≥注册会员40%使用余额 | 余额支付订单÷总订单 |
| 积分兑换率 | ≥注册会员30%使用积分 | 积分抵扣订单÷总订单 |

## 5. 数据约束

### 5.1 数据库约束

| 约束 | 说明 | 理由 |
|:-----|:-----|:-----|
| phone唯一索引 | `member.phone` UNIQUE | 手机号即会员账号，不可重复 |
| 金额单位 | 所有金额以分存储(整数) | 防浮点精度问题 |
| 积分非负 | `member.points >= 0` CHECK约束 | 不允许透支 |
| 余额非负 | `member.balance >= 0` CHECK约束 | 不允许欠款 |
| 等级流转 | 仅允许升序: regular→silver→gold→diamond | 等级保护 |
| 流水不可逆 | 积分/余额流水行只追加，不允许修改/删除 | 审计追溯 |
| 手机号加密 | 写入时加密，读取时解密(应用层) | 数据合规 |
| 软删除 | 会员不做物理删除，设 status=disabled | 数据保留 |

### 5.2 关键数据实体

```typescript
interface Member {
  id: string;
  phone: string;                    // 加密存储
  name: string;
  level: 'regular' | 'silver' | 'gold' | 'diamond';
  points: number;                   // 当前积分(>=0)
  balance: number;                  // 余额(分, >=0)
  totalSpent: number;               // 累计消费(分)
  status: 'active' | 'disabled';
  createdAt: Date;
  expiredAt?: Date;
}

interface PointsTransaction {
  id: string;
  memberId: string;
  type: 'earn' | 'redeem' | 'expire' | 'admin_adjust';
  amount: number;                   // 正=增加 负=扣减
  balanceAfter: number;             // 变动后余额
  orderId?: string;
  remark: string;
  createdAt: Date;
}

interface BalanceTransaction {
  id: string;
  memberId: string;
  type: 'recharge' | 'payment' | 'refund' | 'admin_adjust';
  amount: number;
  balanceAfter: number;
  orderId?: string;
  paymentMethod?: string;           // 仅充值时有值
  createdAt: Date;
}
```

## 6. 不在此PRD范围

- 小程序商城(商品浏览/下单) → 归 Storefront 模块
- 会员之间的积分转移/赠送 → v2考虑
- 企业客户批量注册 → 归 Merchant 模块
- 营销活动(每日签到积分/消费满减) → 归 Coupon/Promotion 模块
