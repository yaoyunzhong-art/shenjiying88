## 1. 目标

### 1.1 业务目标
搭建店A财务对账系统，实现每日营收自动汇总、退款线上化、支付渠道对账，替代店长当前手工记账方式。

### 1.2 产品目标
- 每日期日报自动生成(凌晨00:05跑批)
- 店长查看昨日营收≤5秒
- 支持全单/部分退款操作
- 微信支付渠道自动对账
- 异常交易标记

### 1.3 成功指标
| 指标 | 目标值 | 测量方式 |
|:----|:------|:---------|
| 日结生成时效 | 00:05前完成 | 任务日志 |
| 营收查询响应 | ≤5秒(含汇总计算) | APM |
| 退款处理时长 | ≤10秒/单(从点击到完成) | 端到端 |
| 对账差异检出率 | 100%(如有差异必检出) | 人工复核 |
| 误差率 | 系统日结=实际流水(0误差) | 核对 |

## 5. 数据约束

### 5.1 数据库约束

| 约束 | 说明 | 理由 |
|:-----|:-----|:-----|
| 日报日期唯一 | `daily_report.date` UNIQUE，一天一条 | 防重复跑批 |
| 金额精度 | 所有金额以分存储(整数) | 防浮点精度 |
| 退款不可重复 | 订单状态=refunded 的不可再次退款 | 业务规则 |
| 退款金额约束 | 部分退款不可反向操作调整 | 防篡改 |
| 不可逆操作 | 退款一旦提交不可撤回，需人工处理 | 防资损 |
| 审计只追加 | audit_log表只INSERT不UPDATE/DELETE | 审计合规 |
| 日报不重算 | 历史日报生成后不因后续退款而修改 | 财务规范 |
| 软删除 | 订单/日报不做物理删除 | 数据保留 |

### 5.2 关键数据实体

```typescript
interface DailyReport {
  date: string;                          // yyyy-MM-dd
  totalRevenue: number;                  // 分
  totalRefund: number;                   // 分
  netRevenue: number;                    // 分
  orderCount: number;
  paymentBreakdown: {
    wechat: number;                      // 分
    balance: number;                     // 分
    cash: number;                        // 分
  };
  refundCount: number;
  abnormalOrderIds: string[];            // 异常订单ID列表
  generatedAt: Date;
}

interface MonthlyReport {
  yearMonth: string;                     // yyyy-MM
  totalRevenue: number;
  totalRefund: number;
  netRevenue: number;
  orderCount: number;
  avgDailyRevenue: number;
  paymentPercentage: { wechat: number; balance: number; cash: number };
  generatedAt: Date;
}

interface Refund {
  id: string;
  orderId: string;
  amount: number;                        // 分
  reason: string;
  operatorId: string;
  approverId?: string;                   // >¥500 时必填
  status: 'pending_approval' | 'approved' | 'completed' | 'rejected';
  createdAt: Date;
  completedAt?: Date;
}

interface ReconciliationReport {
  id: string;
  date: string;
  platform: 'wechat' | 'alipay';
  platformTotal: number;
  systemTotal: number;
  diff: number;
  mismatchedOrderIds: string[];
  status: 'matched' | 'mismatched';
  createdAt: Date;
}
```
