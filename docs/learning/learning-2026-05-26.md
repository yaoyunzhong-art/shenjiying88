## 四、学习要点

### 4.1 NestJS 安全最佳实践

1. **Helmet** — HTTP headers 安全（XSS/点击劫持/嗅探）
2. **CORS 严格配置** — 非 `*` origin，白名单域名
3. **Rate Limiting** — `@nestjs/throttler` 或自定义 Guard
4. **Input Validation** — `class-validator` + `ValidationPipe`
5. **Data Masking** — 拦截器层统一脱敏
6. **Audit Log** — 中间件拦截所有写操作并记录

### 4.2 PostgreSQL 连接池优化

```env
# 标准配置
DATABASE_URL="postgresql://user@host/db?connection_limit=20&pool_timeout=10"

# 集成测试专用
DATABASE_URL="postgresql://user@host/db_test?connection_limit=5"
```

- 连接池大小 = (CPU核心数 × 2) + (磁盘数) 
- Jest 并行 worker 数不能超过连接池上限
- Prisma: `connection_limit` query string 参数

### 4.3 大规模 Service 重构模式

```
优先级: 类型外移 > 常量外移 > Helper 提取 > 子服务拆分 > 完全重写
安全度: 类型外移(零风险) > 常量外移(低风险) > Helper(中风险) > 子服务(高风险)
速度:   CSS迁移(3-4min/批次) > 类型外移(2-3min/模块) > Helper(5-8min/个)
```
