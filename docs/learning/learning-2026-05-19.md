## 一、Prisma ORM 生产优化 2026 (前沿追踪)

### Prisma 7 新能力 (搜索自 hirenodejs.com)
| 特性 | 说明 | 对 SPORTS ANT 影响 |
|------|------|-----|
| **Query Insights** | Prisma Postgres 内置慢查询分析 | 可考虑迁移到 Prisma Postgres |
| **Driver Adapters** | 直接连接池, 跳过 pgbouncer | 减少中间层延迟 |
| **Connection Pooling** | `connection_limit` 参数 | 当前未配置, 建议加 |
| **Interactive Transactions** | 更优的批量操作 | 已在部分模块使用 |

### 我们的实践 vs 推荐
| 项目 | 我们 | 推荐 | 差距 |
|------|------|------|------|
| 连接池 | 默认 | 显式 connection_limit=10 | ⚠️ |
| 慢查询 | 无监控 | Prisma Query Insights | ⚠️ |
| N+1 问题 | 手动 include | `findMany({ include })` 已用 | ✅ |
| 批量操作 | `createMany` | `createMany({ skipDuplicates: true })` | → |
| 懒加载 | getter 模式 (2行/文件) | 懒加载是反模式, 应 DI | ⚠️ 已知问题 |

### 新铁律候选
- **Prisma 连接池**: 生产必须 `connection_limit=N`, N = CPU核心数 × 2
- **懒加载迁移**: getter 模式虽快但不利于测试 mock, 建议渐进式迁移到 DI

### 4 道 Gate 设计
```
Gate 1: npx vitest run         → 0 fail 才放行
Gate 2: check-test-compliance  → 集成测试合规检查
Gate 3: verify-test-contracts  → 契约验证 (28/28)
```

### 教训

- 铁律: **husky hook 中的所有命令必须用完整路径或 npx 前缀**
