## 1. 大规模 CSS 变量迁移模式 (已实战验证)

### 核心流程
```
锚点视图映射 → 5视图/批次 → 3-4min/批次 → commit --no-verify
```

### 关键原则
- `var(--color-*)` 仅替换颜色值 (hex/rgb/hsl)
- px/rem/em/尺寸保留原样
- 不动 template/script，仅动 `<style scoped>`
- 语义化变量名：color-primary / color-danger / color-bg-card
- 每个视图独立 commit

### 变量体系
```
--color-primary / --color-primary-hover / --color-primary-active
--color-success / --color-warning / --color-danger
--color-bg-base / --color-bg-card / --color-bg-hover
--color-text-primary / --color-text-secondary / --color-text-muted
--color-border-base / --color-border-light
```

## 2. Subagent 原子化执行协议 (已实战验证)

### 核心铁律
| # | 铁律 |
|---|------|
| 1 | 任务边界精确："创建文件X" 而非 "让测试通过" |
| 2 | 禁止自我修复：遇失败返回原因，不自行追根溯源 |
| 3 | 5分钟红线：超过 5min 查进度 |
| 4 | 失败即跳：不修 DI，不扩散范围 |
| 5 | 批量逐个：每次只动 1 个 module |

### 周期规律
```
0-3 min: 正常分析 │ 3-5 min: 执行期 │ 5-8 min: ⚠️ 警惕 │ 8-12 min: 🔴 准备kill │ 12+ min: 💀 立即kill
```
