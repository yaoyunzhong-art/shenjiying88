## 三、最新技术方案研究 (2025-2026)

### 1. W3C Design Tokens (DTCG Format)
**解决**: P1 #4 设计tokens双源问题

W3C Design Tokens Community Group 已发布标准 DTCG 格式。核心理念：
```
单一JSON源 → Style Dictionary / Cobalt 编译 → CSS变量 + JS常量 + 文档
```
- **Style Dictionary v4**: 支持DTCG格式，可编译到CSS/JS/SCSS/JSON
- **Tokens Studio**: Figma插件 + GitHub同步，设计师直接管理tokens
- 对于Sports Ant：将 design-tokens 迁移到 `tokens.json` → 自动生成 CSS + TS

### 2. CSS @layer 级联分层
**解决**: 品牌覆盖的优先级控制

```css
@layer base, tenant, overrides;
/* base: design-tokens.css → tenant: useBranding动态注入 → overrides: Element Plus覆盖 */
```
相比当前用 `!important` 和 specificity hack，`@layer` 是声明式的、可预测的。

### 3. Playwright Fixtures + Multi-Tenant Testing
**解决**: P1 #6 E2E覆盖不足

```typescript
// 定义租户fixture
const tenantFixtures = base.extend<{ tenantId: string }>({
  tenantId: ['houston', { option: true }],
})

// 参数化所有测试跑10个租户
test.describe('Admin Dashboard', () => {
  for (const t of TENANTS) {
    test(`租户 ${t.name} - 仪表盘渲染`, async ({ page }) => {
      await page.goto(`/${t.id}/dashboard`)
    })
  }
})
```

### 4. NestJS Guard + Rate Limiting (2026 Pattern)
**解决**: P0 #2 无API认证/限流

```typescript
// JWT Guard per-tenant
@UseGuards(TenantAuthGuard, ThrottlerGuard)
@Throttle({ default: { limit: 100, ttl: 60000 } })
@Controller('api/v1/:tenantId')
```
NestJS 11 内置 `@nestjs/throttler` 支持 Redis 分布式限流，可做到租户级别。

### 5. Prisma + PgBouncer 连接池
**解决**: 多租户连接爆炸

当前 schema 无连接池配置。10租户 × N并发 = 连接耗尽风险。
- **PgBouncer** (transaction mode): 前端连接池，已有10年成熟方案
- **Prisma Accelerate**: Prisma 官方连接池服务 (商业化)
- 建议用 PgBouncer Docker sidecar

### 6. pnpm Catalogs (pnpm 9+)
**解决**: monorepo 依赖版本一致性

```yaml
# pnpm-workspace.yaml
catalog:
  vue: ^3.5.0
  '@nestjs/common': ^11.0.0
```
所有子包引用 `"vue": "catalog:"` 替代硬编码版本号。防止版本漂移。

### 7. CSS Nesting + View Transitions API
**解决**: CSS可维护性 + UX

- **CSS Nesting**: 原生浏览器支持（2024+），不需要预处理器即可写嵌套规则
- **View Transitions API**: SPA页面切换动效，`document.startViewTransition()` 一行代码

### 8. OpenTelemetry 分布式追踪
**解决**: 微服务可观测性

```typescript
// NestJS OpenTelemetry interceptor
import { OtelModule } from 'nestjs-otel'
```
追踪从 API Gateway → 各 backend service 的完整请求链路。
