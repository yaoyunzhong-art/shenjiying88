## 一、C端体验四项核心原则的可落地开发约束

### 1.1 便捷原则

| 约束项 | 规格值 |
|--------|--------|
| 核心操作步数 | ≤3步 (预约、下单、兑换、分享等) |
| 关键信息展示 | 一屏展示，无需滚动即见核心内容 |

**可落地约束:**
- 所有C端页面（预约页、商品详情、权益中心、活动页面）必须通过UI设计评审确保主路径≤3步
- 首页/核心页面在 375px 宽度下一屏可视区内展示：核心操作入口、关键数据（余额/积分/等级）、当前权益
- 每一步操作必须有明确的进度指示器（Step Indicator / Breadcrumb）
- 懒加载/骨架屏仅在非核心区域使用，关键信息区禁止占位

### 1.2 科学原则

| 约束项 | 规格值 |
|--------|--------|
| 营销触达AI依据 | 100% 营销推送必有AI决策依据记录 |
| 推荐匹配度 | >80% |

**可落地约束:**
- 每次营销推送前必须经过 AI 决策引擎，决策记录写入 `marketing_push_decision_log`
- 推荐匹配度算法: 客户画像特征与推送内容标签的余弦相似度 ≥ 0.80
- 匹配度低于80%的推送被引擎自动拦截，并记录拦截原因
- 所有 AI 决策必须可追溯（决策ID、时间、画像快照、匹配分数、算法版本）

### 1.3 少打扰原则

| 约束项 | 规格值 |
|--------|--------|
| 每日营销推送上限 | 每客户 ≤1 条 (P2+P3合计) |
| 免打扰时段 | 23:00-08:00 静默P2/P3 |
| 一键关闭营销 | 客户可一键关闭所有P3级推送 |

**可落地约束:**
- 频率计数器基于 `(tenant_id, member_id, date)` 聚合，Redis 原子递增
- 免打扰时段在推送调度层做硬拦截，P2/P3 推送队列在 23:00-08:00 不发送
- 一键关闭: 在客户偏好设置页提供 "关闭全部营销推送" 开关，写入 `member_push_preference.marketing_optout = true`
- 营销推送发送前，中间件检查 optout 标记——已关闭的直接跳过

### 1.4 AI精准营销原则

| 约束项 | 规格值 |
|--------|--------|
| 营销转化率提升 | ≥30% (相对基准) |
| 客户投诉率下降 | ≥50% (相对基准) |

**可落地约束:**
- 建立转化率基线（基于推送后7天内转化事件），AI持续优化
- 投诉/退订作为负反馈进入模型训练
- 月度看板自动对比基线 vs AI决策推送的转化率、投诉率差异
- 连续2个月未达标的自动触发告警 + 模型回滚机制

## 二、全渠道触达体系开发约束

### 2.1 P0-P3四级推送分级 (16.5)

| 级别 | 定义 | 渠道策略 |
|------|------|----------|
| **P0 交易类** | 消费确认、退款、合同变更、法律通知 | 邮件(必发) + App推送/微信服务通知 + 短信(备选) |
| **P1 服务类** | 预约确认、权益变动、到期提醒 | App推送/微信服务通知(主) → 短信(辅) → 邮件(保底) |
| **P2 权益类** | 月度账单、升级通知、积分变动 | App推送/微信服务通知(主) → 邮件(保底) |
| **P3 营销类** | 活动促销、新品推荐、品牌传播 | App推送/微信服务通知(主) → 社媒/公众号(辅) |

**可落地约束:**
- 推送记录表 `push_record` 必须包含 `push_level` 字段 (enum: P0|P1|P2|P3)
- 渠道路由引擎根据 push_level + 渠道健康度 + 成本评分 + 客户偏好 综合决策
- P0/P1 不可被客户关闭；P2/P3 可关闭

### 2.2 邮件作为底层逻辑 (16.2)

**必须发送邮件(不可关闭)的场景:**
- 交易记录 (消费确认、退款、储值充值) — P0
- 合同变更 (会员协议、隐私政策、服务条款) — P0
- 权益变动 (等级变更、SVIP变更、积分重大变动) — P0
- 法律通知 (数据泄露、服务暂停、账户异常) — P0
- 预约确认 (成功/变更/取消) — P1
- 重要权益提醒 (积分到期、券到期、SVIP到期)前7/3/1天 — P1
- 生日趴确认 — P1
- 团建合同签署 — P0
- 联盟分账月报 — P1

**可选关闭:**
- 月度账单 — P2 (可关闭)

**可落地约束:**
- 必须集成邮件发送服务（可选 SendGrid / AWS SES / 阿里云邮件）
- 海外 P0/P1 邮件用事务性模板（transactional template）防止进垃圾箱
- 海外邮件改用当地邮件服务商（地域化发送）
- 退订确认页展示退订详情，提供 "一键恢复订阅" 按钮
- 邮件发送失败必须降级到短信（P0）或记录告警（P1-P3）

### 2.3 短信双通道自动切换

- 集成阿里云短信 + 腾讯云短信
- 主通道发送失败率 > 5% 时自动切换备通道
- 健康度探针每5分钟检测双通道可用性

### 2.4 AI智能渠道决策引擎 (16.6)

- 渠道选择算法: 根据 push_level + 渠道历史打开率 + 渠道成本 + 用户偏好 + 实时健康度
- 渠道组合策略: 可组合多通道（主+辅+保底）
- 效果追踪: 每次推送追踪送达率、打开率、转化率
- **V5.1优化** 渠道健康度评分: 根据历史送达率和打开率动态调整渠道权重
- **V5.1优化** 渠道性价比评分: 综合成本与转化率进行决策

### 2.5 客户全渠道偏好设置 (16.7)

| 渠道 | 可关闭范围 | 备注 |
|------|-----------|------|
| 邮件 | 营销类(P3)可关闭 | 交易/服务/权益类不可关闭 |
| 短信 | P0/P1可关闭 | 验证码除外 |
| App推送 | P2/P3可关闭 | P0/P1不可关闭 |

**一键全关闭营销** → 关闭所有 P3 级推送（所有渠道）

## 三、C端营销推送全局管控约束 (第17章)

### 3.1 频率管控 (17.1)

| 维度 | 上限 |
|------|------|
| 单客户每日 P2+P3 | ≤1 条 |
| 单客户每周 P3 | ≤3 条 |

**可落地约束:**
- Redis + Lua 脚本实现原子计数
- `daily_push_count:{tenant}:{member}:{date}` —— 每日计数器
- `weekly_push_count:{tenant}:{member}:{week_number}` —— 每周计数器（仅P3）
- 超过上限的推送请求直接拒绝并记录到 `push_denied_log`

### 3.2 免打扰时段 (17.1)

- 23:00-08:00 所有 P2/P3 静默
- P0/P1 不受免打扰限制
- 时段配置可系统级调整（通过配置中心下发）

### 3.3 会员差异化推送 (17.2)

| 会员等级 | P2/P3频率 | 推送侧重 |
|----------|----------|----------|
| 青铜-白银 | 每日≤1条 | 升级引导、新人福利、基础活动 |
| 黄金-铂金 | 每日≤1条 | SVIP引导、专属活动、高阶特权 |
| 钻石-王者 | 每3天≤1条 | 专属活动、新品内测、生日关怀 |
| SVIP会员 | 每周≤2条 | SVIP专属活动、新品优先购 |

### 3.4 营销疲劳度保护 (17.3)

- **疲劳度评分:** AI根据近7天接收数、打开率、退订率计算（0-100分）
- **分层措施:**
  - 疲劳度 > 70: 降为每3天1条
  - 疲劳度 > 90: 暂停P2/P3推送7天
- 疲劳度评分写入 `member_fatigue_score` 表，每小时/事件触发更新

### 3.5 自动效果追踪 (17.4)

- 每次推送追踪: 打开率、转化率、退订率
- **自愈规则:**
  - 打开率 < 10%: 该推送模板自动下架
  - 退订率 > 1%: 该活动自动暂停
- 全量推送记录写入 `push_event_log`，支持多维分析看板

## 四、AI精准营销引擎 (第18章)

### 4.1 客户画像持续学习 (18.1)

**画像维度:**
- 消费偏好、价格敏感度、活跃时段、渠道偏好
- 营销疲劳度、生命周期阶段、会员等级
- 推送偏好、社交偏好

**可落地约束:**
- 实时更新: 每次客户行为事件触发画像更新
- 所有营销决策基于最新画像快照（禁止使用过期画像）
- 画像数据存储: `member_profile` 表，JSONB格式可扩展

### 4.2 营销时机AI决策 (18.2)

- **最佳时机模型:** AI学习每个客户历史打开率与推送时间的关系
  - 输入: 客户历史推送事件的 (hour, dayOfWeek, opened) 三元组
  - 输出: 预测的最佳推送时间窗口
- **场景触发列表:**
  - 到店触发 → 到店有礼 / 新品推荐
  - 消费后触发 → 下次优惠 / 会员升级引导
  - 排队中触发 → 闲时优惠推荐
  - 离店后触发 → 评价邀约 / 复购唤醒
  - 沉睡唤醒 → 休眠 > 30天唤醒
  - 生日前30天 → 生日趴引导
  - 团建后7天 → 团建报告 + 下次推荐
  - 联盟券推荐 → 异业联盟券精准推送

### 4.3 营销内容AI生成与匹配 (18.3)

- **内容库:** 基于营销知识库(RAG)的模板库
- **个性化匹配:** 根据画像从内容库选取匹配度 > 80% 的内容
- **动态生成:** AI生成个性化文案（标题 + 正文 + CTA）
- **A/B测试:** 每次推送可选A/B变体，自动优选出转化率高的变体

### 4.4 协同过滤算法 (V5.1优化)

- 根据相似客户的偏好推荐内容
- 相似度算法: User-based CF / Item-based CF / 混合模型
- 冷启动: 新客户使用基于标签的推荐

### 4.5 营销效果AI评估与迭代 (18.4)

- 实时监控: 打开率、转化率、退订率
- **AI每周输出:** 《营销效果周报》，包含:
  - 各活动ROI排名
  - 各渠道效果对比
  - 客户疲劳度趋势
  - 优化建议
