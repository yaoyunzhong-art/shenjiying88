# AB Test Bias Pattern (A/B 测试偏差反模式)

> **版本**: v4 · **创建**: 2026-06-28 · **来源**: Phase-42 T172

A/B 测试是营销决策的科学依据, 但若忽视以下 5 种偏差, 实验结论将误导业务。

## 5 反模式

### 1. 样本偏差 (Sample Bias)
- **症状**: 实验组和对照组人群画像差异巨大
- **原因**: 未使用 hash 分流, 直接用 memberId % 2
- **修复**: djb2 hash → 50/50 分配, 锁定实验周期

### 2. 时间偏差 (Time Bias)
- **症状**: 工作日和周末数据混合
- **原因**: 实验周期包含节假日
- **修复**: 完整周期 ≥ 7d, 排除节假日, 同期对照

### 3. 早停偏差 (Peeking Bias)
- **症状**: 显著性未达就提前结束
- **原因**: PM 看到 A 略好就宣布胜出
- **修复**: `canStopEarly()` 强制 p < 0.05 才允许结束

### 4. 网络效应 (Network Effect)
- **症状**: 同群用户互相影响
- **原因**: 团购/社交场景下用户间干扰
- **修复**: 地理/聚类隔离 (cluster randomization)

### 5. 选择偏差 (Selection Bias)
- **症状**: 高活跃用户被优先纳入
- **原因**: 实验入口过滤了部分用户
- **修复**: 全量随机, 事后按分层分析

## 5 清单

- [ ] 实验必须 hash 分流 (djb2 / FNV)
- [ ] 最小样本 1000 + p < 0.05 才发布结论
- [ ] 显著性未达禁止早停
- [ ] 完整周期 7d+ 排除节假日
- [ ] 地理/聚类隔离避免网络效应

> 🦞 "A/B 测试 = 科学决策, 偏差 = 错误的科学"
