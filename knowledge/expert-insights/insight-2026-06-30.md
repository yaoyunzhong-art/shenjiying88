## E23: 跨模块 E2E 链扩展经验 — Mobile反向链路·企业配额·数据管道 (Pulse-Nightly-06)

### 本次扩展的 3 条新链

| 链 | 路径 | 核心验证点 | subtests | 模式 |
|----|------|-----------|:--------:|:----:|
| #10 | Mobile(注册/登录/下单) → API → Domain → Admin | 手机号唯一性·库存校验·地址验证·分页 | 13 | **反向链路 (Mobile起点)** |
| #11 | Tob-Web(门店/员工) → SDK(事件) → API → Domain(配额) → Admin | 门店配额·合约冻结·SDK事件·编码唯一·邮箱/时薪 | 11 | **企业配额+基础设施事件** |
| #12 | Admin(报表) → API → Domain → Storefront → Analytics | 多维聚合·品类占比·降序排序·空数据·时间窗口 | 11 | **数据管道+报表看板** |

### 关键发现

1. **反向链路验证的共享状态风险**: 链10 首次以 mobile App 为起点覆盖 C 端全流程。反向链路在共享仓储环境下需要特别注意状态隔离——手机号重复注册测试和后续测试可能产生干扰。

2. **企业配额系统的 Domain 层防护**: 链11 验证了门店配额上限、合约冻结、员工时薪/邮箱等企业级约束，确认 Domain 层作为"业务防火墙"的有效性。SDK 事件日志证明基础设施层在业务事件后可追踪。

3. **数据管道测试的维度聚合验证**: 链12 首次引入多维度报表聚合（门店排行为降序、品类占比总和≈100%、销售趋势点按日期排序）。验证了数据从原始点到 KPI 看板的完整 ETL 路径。

4. **覆盖 12 链后的缺口分析**: 现有 12 链中串行流程覆盖完整，但缺少：
   - **并发场景**: 无多端同时操作验证
   - **大数据量**: 无 365+ 天报表/10 万+ 条数据场景
   - **真实 HTTP 集成**: 全部使用 in-memory 模拟

### 新增的 E2E 模式

三条新链引入以下模式:
- **Mobile C端全流程反向链路**: 用户注册→登录→下单→Admin 核验（反向·数据流回 admin）
- **企业配额与基础设施事件模式**: Domain 配额拦截 + SDK 队列事件 + Admin 汇总视图
- **数据管道与多维聚合模式**: 原始数据 → Domain 聚合（品类/门店/趋势） → Analytics KPI 看板
