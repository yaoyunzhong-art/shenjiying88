## T4: 情感累积模式 — 工程经验

### 问题发现
链38 N4 测试中, `handleCustomerMessage` 的 sentiment 每次被 `newSentiment` 完全覆盖, 导致:
- 第一条投诉消息 → sentiment=negative
- 第二条中性消息 → sentiment 被重置为 neutral
- shouldEscalate(检查 sentiment===negative 且 messages>=4) 始终返回 false

### 修复方案: sentimentPriority 累积
```
const sentimentPriority = { negative: 2, positive: 1, neutral: 0 }
const finalSentiment = sentimentPriority[newSentiment] >= sentimentPriority[currentSentiment]
  ? newSentiment
  : currentSentiment
```
注意: 此方案不降级, 仅升级或保持。如果业务需要可降级(比如道歉后 positive 覆盖 negative), 则改为完全覆盖。

### 可复用场景
- 订单状态机: pending → confirmed → shipped → delivered → cancelled
- 会话状态机: active → escalated → resolved → closed
- 异常等级: critical > warning > info (同样的优先级逻辑)
