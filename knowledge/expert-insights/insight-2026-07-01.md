## E25: 跨模块 E2E 链扩展经验 — 多渠道并发·国际化深度·大数据量 (Pulse-Nightly-07)

### 本次扩展的 3 条新链

| 链 | 路径 | 核心验证点 | subtests | 模式 |
|----|------|-----------|:--------:|:----:|
| #13 | Mobile+Storefront → API → Domain | 库存临界值并发、幂等性、手机号唯一性、大批量并发 | 11 | **多渠道并发一致性** |
| #14 | Miniapp → SDK → API → Domain(国际化) | 6 locale 跨语言订单、语言索引、emoji/特殊字符、空字段 | 22 | **国际化深度 + 跨语言数据一致性** |
| #15 | Admin → API → Domain | 50,000条分页、365天截断、幂等性缓存、10万级估算 | 18 | **大数据量性能基准 + 幂等性深度** |

### 关键发现

1. **并发测试的串行局限**: 链13 通过串行模拟并发操作（逐条处理请求）验证了逻辑层的资源约束，但无法发现物理并发下的竞态条件。Node.js test runner 原生不支持多线程并发测试。解决方案方向：使用 Worker Threads 或独立进程。

2. **国际化测试的 locale 扩展性**: 链14 使用 TypeScript 枚举 + Map 数据结构管理多语言数据。新增 locale 只需在 `LocaleCode` 枚举添加 + 在 `PRODUCT_I18N_MAP` 补充对应语言字符串。6 种 locale 覆盖了东亚和东南亚主要市场，但未覆盖印度(kn-IN)和孟加拉(bn-BD)等南亚区域。

3. **大数据量测试的随机性管理**: 链15 使用 `Math.random()` 生成 50,000 条销售数据，测试验证使用模式断言（>0、降序排列）而非具体值断言。365天截断保护是 Domain 层的关键安全边界。

4. **幂等性多层验证**: 链13 和链15 分别验证了 requestId 级幂等（判断是否已处理）和缓存级幂等（相同报表请求返回缓存结果）。幂等键应跨操作类型隔离（order vs register vs store_create）。

### 新增的 E2E 模式

- **多渠道并发一致性**: 模拟多端同时下单的库存校验和幂等性保障
- **国际化深度与跨语言数据一致性**: 6 locale 独立索引、特殊字符处理、语言回退
- **大数据量性能基准与截断保护**: 万级数据分页、日期范围截断、pageSize/page clamp、性能估算
