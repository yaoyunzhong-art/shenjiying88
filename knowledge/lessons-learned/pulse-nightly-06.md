## 测试设计关键发现

### 1. 反向链路的共享状态问题
链10/11 中多条链共享同一仓储层（USER_STORE/STORE_STORE），导致测试之间的依赖。Node.js `--import tsx --test` 在同一进程中共享模块作用域。解决方案:
- 使用 `Date.now()` 或 `Math.random()` 生成唯一编码前缀
- 在测试中计算 `needCount = maxCount - existingCount` 灵活调整
- 使用断言失败信息携带错误细节以辅助调试

### 2. 测试数据边界值必须精确
链11 的超长门店名称测试中:
- 原始字符串长度 42 或 46 或 55，需要精确计算验证 code 边界
- 手机号格式化测试中，`String(100+i).slice(1)` 在 i=0 时产生 `'00'` 导致只有 10 位（有效11位）
- 教训: 硬编码边界测试值后用 assert 验证边界条件

### 3. 数据管道测试的随机性挑战
链12 使用 `Math.random()` 生成测试数据，导致每次运行结果不同。解决方案:
- 使用随机种子 seed
- 不硬编码具体值，断言模式（如 >0、降序排列、百分比总和≈100%）
- 使用结构验证而非值验证
