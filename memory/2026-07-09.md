# 2026-07-09 验收脉冲笔记

## 🦞 脉冲#246 洞察

### 1. React SSR: renderToStaticMarkup + useState 初始化未过clamp
`ResizablePanel` 组件使用 `useState(defaultRatio)` 初始化，但未使用 `clampFn` 夹值。SSR渲染时直接输出未clamp的初始值（如 `calc(5%`），导致测试断言失败。修复方案: `useState(clampFn(defaultRatio))`。

**教训**: React组件的SSR初始渲染与客户端hydrate时的state必须一致。如果初始化不经过clampFn，SSR输出错误的初始style值，测试会因HTML内容不匹配而失败。

### 2. Node test runner 断言位置错报
Node 24.5 的 test runner 在 `renderToStaticMarkup` SSR 渲染的组件测试中，如果早期 assertion 失败，可能将错误归因到后续（字符索引不同）的 assertion 上。本次 `clamps ratio to minRatio` 测试实际失败在第1个 assertion (`calc(10%`)，但错误消息报告 `html.includes('aria-orientation="horizontal"')`（属于`separator has role and aria attributes`测试的断言）。

**教训**: 当node test runner报告的断言内容与预期行内容不匹配时，应检查是SSR组件前几个assertion中哪个真正失败了，而非盲目寻找错误信息中提到的具体值。
