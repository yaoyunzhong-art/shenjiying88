# 📊 reports — 测试报告目录

## 模块简介

`reports/` 是神机营 SaaS 平台自动化测试体系生成的**测试报告存档目录**，存放每日凌晨执行的 E2E 测试与深度测试输出结果。报告以 Markdown 格式存档，按日期分区命名（`nightly-test-YYYYMMDD.md`），记录每次脉冲执行的测试覆盖、通过率、新增链路和关键发现，形成完整的测试历史追溯链。

当前存档跨度：**2026-06-27 ~ 2026-07-18**，共 14 份日报，由 `testing-system/testing-master.sh` 主控脚本调度产出。

## 报告内容规范

每份报告包含以下几个核心板块：

| 板块 | 内容 | 示例 |
|------|------|------|
| **总体状态** | 执行阶段、新增测试链数、subtests 数、通过率 | 3 链 / 58 个 / 58/58 ✅ |
| **新增 E2E 链** | 每链的路径描述（前端→API→后端→展示），按正例/反例/边界统计 | 链34: Admin活动创建→Storefront活动展示→Miniapp报名 |
| **覆盖率演进** | 累计链数、subtests、覆盖应用、覆盖角色 | 36 链 / ~338 个 / 7 apps |
| **问题记录** | 零日缺陷、回归问题、复现率、临时修正手段 | 链18 退款脚本 token 过期 |
| **复盘改进** | 当天问题的根因分析与自动化改进措施 | 增加 token 预刷新逻辑 |

## 使用方式

```bash
# 查看最新报告
cat reports/nightly-test-$(date +%Y%m%d).md

# 对比两天的测试覆盖演进
diff -u reports/nightly-test-20260717.md reports/nightly-test-20260718.md

# 统计历史通过率
grep -h "通过率\|通过\s*/\s*" reports/*.md | head -20

# 查看某次脉冲的新增链路详情
head -80 reports/nightly-test-20260718.md
```

## 报告生成流程

1. **触发**：`testing-system/testing-master.sh` 调度器在 03:30 CST 启动 Pulse-Nightly 窗口
2. **执行**：运行器按计划执行 `scripts/run-full-deep-test.sh` 全量深度测试
3. **采集**：`monitoring/` 模块实时采集每个测试链的通过/失败状态
4. **生成**：`reporting/` 报告生成器将监控数据渲染为 Markdown 日报
5. **归档**：报告写入本目录，按日期永久存档

## 注意事项

- 报告为自动生成，格式统一，不建议手动修改；如需补充说明，请在末尾追加注释
- 报告文件名中的日期对应执行日期，部分周末可能存在跳档
- 每份报告末附有"复盘改进"板块，记录了当日发现的自动化测试管线优化点
- 历史报告可用于对比趋势，评估测试覆盖率的健康度演进
