## 📌 发现的问题

### 严重问题

1. **测试覆盖率严重不足** — 26 个页面中仅 7 个独立页面有测试（page.tsx + form/new + capability-access + health-score + [id]），其余 18 个子页面全部零测试
2. **大多数子页面规模偏小** — 25 个页面中仅 11 个 >200 行，14 个页面不到 200 行（最小的 audit 仅 98 行），可能功能不完整或 UI 过简

### 其他发现

3. **`capability-access` 是 Server Component** — 使用 `export default async function`，不含 `'use client'`。这是设计意图，但与其他 24 个客户端组件不一致
4. **`[id]/events` 和 `[id]/shift-handover`** — 存在 page.tsx 但不在 26 个目标清单中，属于额外页面
5. **需追加重测的页面** — orders(162行)、members(126行)、inspection(115行) 等功能重要但页面太薄
